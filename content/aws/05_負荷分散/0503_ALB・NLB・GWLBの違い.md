## 補足：ALB・NLB・GWLBの違い（直感的な理解）

### 「何を見て、どこに流すか」だけ覚える

3つの違いは突き詰めると **「パケットのどこを読んで、何をするか」** の違い。

### 郵便物アナロジー

```
GWLB  ─── 「まず全部、税関（セキュリティ検査）に回せ」
             ↑ 宛先すら見ない。とにかく検査を通す係

NLB   ─── 「宛先の住所（IPアドレス・ポート番号）だけ見て振り分ける」
             ↑ 封筒の表だけ見る。中身は開けない

ALB   ─── 「封筒を開けて、手紙の内容（URL・ヘッダー）を読んで振り分ける」
             ↑ 中身まで読んで「これはAさん宛、これはBさん宛」と判断
```

### 「なぜそれが必要か」で覚える

**ALB が嬉しい場面**
→ `/api` と `/web` で別サーバーに分けたい。URLを見ないと振り分けられない。だからL7まで読む。

**NLB が嬉しい場面**
→ ゲームサーバーやIoTで「とにかく速く・大量に」流したい。URLなんか読んでる暇はない。バーコードだけ見て即転送。あと固定IPがほしい（ファイアウォールのホワイトリスト登録のため）。

**GWLB が嬉しい場面**
→ 大企業で「全通信を必ずPalo Altoのファイアウォールに通せ」という要件がある。でもファイアウォールが1台だと壊れたとき困る。GWLBがファイアウォール複数台を束ねて冗長化してくれる。

### SAA的に最低限覚える3行

- **ALB** = URLの中身を読める → パスベースルーティング・Lambda対応・WAF
- **NLB** = 読まないから速い → UDP・固定IP・超低遅延
- **GWLB** = セキュリティ機器を「透過的に」束ねる専用品
