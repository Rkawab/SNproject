# S3（Simple Storage Service）

**S3 = 容量無制限・高耐久のオブジェクトストレージ**。
ファイル（オブジェクト）を「バケット」に入れ、HTTP/HTTPS（API・URL）でアクセスする。サーバーにマウントして使うディスクではない。

## 基本概念

| 用語 | 意味 |
|---|---|
| **バケット** | オブジェクトを入れる最上位の入れ物。**名前はグローバルで一意**（世界中で重複不可） |
| **オブジェクト** | 保存される実体（ファイル＋メタデータ）。**1オブジェクト最大5TB** |
| **キー** | バケット内でのオブジェクトの名前（`photos/2024/cat.jpg` のようなパス形式） |
| **プレフィックス** | キーの先頭部分（フォルダ風に見えるが、実際はフラットな名前空間） |

## S3の特性

| 項目 | 内容 |
|---|---|
| 耐久性 | **イレブンナイン（99.999999999%）**。複数AZに自動複製 |
| 可用性 | 99.99%（Standardクラス） |
| 容量 | 無制限（事前確保不要・使った分だけ課金） |
| 所属 | **リージョンサービス（VPCの外）**。VPC内からは VPCエンドポイント経由でアクセス可能 |
| 整合性 | 強い整合性（書き込み直後の読み取りで最新が返る） |

## 主な用途

| 用途 | 具体例 |
|---|---|
| 静的ファイル配信 | WebサイトのHTML・画像・動画 → [[0108_EC2とS3のWebサイトの違い]] |
| バックアップ置き場 | EBSスナップショットの保存先 |
| データレイク | ML・分析用データの蓄積（Athena等から直接クエリ可） |
| ログ保存 | CloudTrail・ALBのログ |
| ユーザーファイル | アプリにアップロードされた画像など |

## EBS・EFSとの違い（概要）

| サービス | ストレージ種別 | アクセス方法 |
|---|---|---|
| EBS | ブロック | EC2にマウント（1台） |
| EFS | ファイル（NFS） | 複数EC2からマウント |
| **S3** | **オブジェクト** | **HTTP API・URL（マウント不要）** |

詳細比較 → [[0202_S3とEBS・EFSの違い]] / [[0103_EC2のストレージ]]

## SAA試験の判断フロー

```
データをどう使いたい？
├── EC2のディスクとして使いたい        → EBS
├── 複数EC2で共有したい（Linux）       → EFS
├── 複数EC2で共有したい（Windows）     → FSx for Windows
├── URLやAPIでアクセスしたい           → S3
└── 超高速な一時データ                 → インスタンスストア
```
