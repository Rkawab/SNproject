## S3バージョン管理（Versioning）

S3バケットに保存されるオブジェクトの**すべての変更履歴を保持**する機能です。

### 基本的な仕組み

バージョン管理を有効にすると、同じキー名のオブジェクトを上書き・削除しても、過去のバージョンがすべてバケット内に残ります。各バージョンには一意の**バージョンID**が付与されます。

```
s3://my-bucket/photo.jpg
  ├── バージョンID: abc123  ← 最新
  ├── バージョンID: def456
  └── バージョンID: ghi789  ← 最古
```

### 削除の動作

バージョン管理が有効な場合、通常の削除操作ではオブジェクトは**完全には消えません**。

- 通常削除 → **削除マーカー（Delete Marker）** が作成されるだけ。過去バージョンは残る
- 完全削除 → バージョンIDを指定して削除する必要がある

### 有効化の状態遷移

バージョン管理には3つの状態があります。

|状態|説明|
|---|---|
|**未設定（Unversioned）**|デフォルト。バージョン管理なし|
|**有効（Enabled）**|全オブジェクトのバージョンを保持|
|**一時停止（Suspended）**|新規オブジェクトはバージョン管理なし。既存バージョンは保持|

⚠️ 一度有効にすると「未設定」には戻せません（一時停止まで）。

### 主なユースケース

- **誤操作からの復旧** — 誤って上書き・削除したファイルを復元
- **データ保護** — ランサムウェア対策（MFA Delete併用で強化）
- **監査・コンプライアンス** — 変更履歴の追跡

### コスト面の注意

すべてのバージョンが保存されるため、**ストレージコストが増加**します。不要なバージョンを自動削除するには**ライフサイクルポリシー**との併用が推奨されます。

## S3 Object Lock（WORM保持）

**S3 Object Lock** は、一定期間オブジェクトの削除・上書きを防ぐ機能です。WORM（Write Once Read Many：一度書いたら何度も読めるが変更できない）要件、監査ログ、金融・医療などの規制対応で出ます。

### バージョニングとの関係

- Object Lock を使うバケットでは **バージョニングが有効** になる
- 保持対象は「オブジェクトのバージョン」単位
- 通常のバージョニングは「消しても戻せる」、Object Lock は「保持期間中は消せない/上書きできない」

### 保持モード

| モード | 変更権限 | 使いどころ |
|---|---|---|
| **Governance mode** | 特別な権限があれば保持設定を上書き可能 | 社内統制・誤削除防止 |
| **Compliance mode** | rootユーザーを含め、保持期間中は上書き・削除不可 | 法規制・厳格なWORM要件 |

> [!tip] SAAひっかけ
> 「削除を検出」ならイベント通知やCloudTrailでもできるが、「削除・上書きを防止」なら **S3 Object Lock**。
> 「誤削除から復元」ならバージョニング、「保持期間中に誰にも消させない」なら Object Lock。

## Gitとの比較で理解する

| |Git|S3バージョン管理|
|---|---|---|
|履歴保持|コミット履歴|バージョンID|
|削除|ファイル削除のコミット|削除マーカー|
|復元|`git checkout`|バージョンIDを指定して取得|
|差分管理|あり（diff）|なし（スナップショットのみ）|

### 主な違い

**Gitと違う点**
- **差分ではなくフルコピー保存** — Gitは差分管理ですが、S3は変更のたびにオブジェクト丸ごと保存します。なので大きなファイルを頻繁に更新するとコストが跳ね上がります
- **ブランチ概念はない** — 単純な「時系列の積み重ね」だけです
- **自動プッシュ** — S3に書き込んだ瞬間に新バージョンが作られます。コミットのような明示的な操作は不要

## SAA試験ポイント

| 問われ方 | 答え |
|---|---|
| 誤って上書き・削除したオブジェクトを復元したい | **バージョニング有効化** |
| 削除をさらに厳重に保護したい | **MFA Delete**（バージョン完全削除にMFA必須化） |
| 一度有効にしたバージョニングを「未設定」に戻せる？ | **戻せない**（Suspended＝一時停止まで） |
| 通常削除すると何が起きる？ | **削除マーカー**が付くだけで過去バージョンは残る |
| WORM要件で保持期間中の削除・上書きを防ぎたい | **S3 Object Lock** |
| 古いバージョンのコストを抑えたい | **ライフサイクルポリシー**で非現行バージョンを自動削除/IA移行 → [[0204_S3のライフサイクル管理]] |
| レプリケーションの前提条件 | **送信元・送信先両方でバージョニング有効** → [[0206_S3のレプリケーション]] |
