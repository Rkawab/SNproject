# Amazon FSx ファミリー4種の使い分け

> [!info] このノートの位置づけ
> [[0202_S3とEBS・EFSの違い]] では「FSx for Windows File Server」だけ触れたが、SAAでは **FSxは4兄弟**で出題され、特に **Lustre** は頻出。ここで4種の見分け方を1枚にまとめる。

## FSxとは

**フルマネージドの「ファイルシステム貸し出しサービス」**。EFSがNFS一種類なのに対し、FSxは**既存の有名ファイルシステムをそのままマネージドで提供**する。

> [!tip] たとえ話：レンタル倉庫チェーン
> EFSが「標準的な貸し倉庫（Linux向けNFS）」だとすると、FSxは「**お客の流儀に合わせた特殊倉庫チェーン**」。
> Windows派にはWindows仕様の倉庫、研究所（HPC）には超高速搬出入の倉庫、NetApp利用企業には使い慣れたNetApp仕様の倉庫を用意してくれる。

---

## 4兄弟の見分け方（ここだけ覚えれば解ける）

| 種類 | ひとことで | 問題文のキーワード（これが出たら反射） |
|---|---|---|
| **FSx for Windows File Server** | Windowsの共有フォルダ | **SMB / NTFS / Active Directory（AD）連携 / Windowsアプリ** |
| **FSx for Lustre** | HPC向け超高速並列ファイルシステム | **HPC / 機械学習の学習データ / 動画レンダリング / ミリ秒未満・高スループット / S3と連携** |
| **FSx for NetApp ONTAP** | NetApp倉庫のマネージド版 | **オンプレのNetApp（ONTAP）から移行 / NFSとSMBの両対応 / SnapMirror** |
| **FSx for OpenZFS** | ZFS倉庫のマネージド版 | **オンプレのZFSサーバーから移行 / NFSで低レイテンシ** |

> [!tip] 覚え方
> - **Windows** と書いてあれば Windows File Server（そのまま）
> - **速さ・計算・研究**っぽければ Lustre（Linux + Cluster が語源＝計算クラスタ用）
> - **NetApp / ZFS** は固有名詞そのまま＝「**今それを使っている会社の引っ越し先**」

---

## 頻出2種の詳細

### FSx for Windows File Server

- **SMBプロトコル**のファイル共有。**NTFSアクセス権**・**Active Directory連携**をそのまま使える
- 用途：Windowsアプリのファイル共有、ホームディレクトリ、オンプレWindowsファイルサーバーの移行先
- マルチAZ配置で高可用性に対応

### FSx for Lustre

- **HPC（ハイパフォーマンスコンピューティング）向け**の並列分散ファイルシステム。数百GB/s級のスループット・ミリ秒未満のレイテンシ
- **S3とネイティブ連携**できるのが試験の急所：S3バケットをFSx for Lustreにリンクすると、S3のデータを**高速ファイルシステムとして処理**し、結果をS3へ書き戻せる
- 用途：機械学習の学習ジョブ、金融シミュレーション、動画レンダリング、ゲノム解析

```
S3（安い置き場）⇄ FSx for Lustre（高速作業台）⇄ EC2群（計算クラスタ）
　→「データはS3に貯めて、計算のときだけLustreで高速処理」が定番構成
```

## EFS vs FSx の決着表

| 要件 | 答え |
|---|---|
| **Linux**の複数EC2で共有（一般用途） | **EFS** |
| **Windows** / SMB / NTFS / AD | **FSx for Windows File Server** |
| **HPC・ML・超高速**・S3のデータを高速処理 | **FSx for Lustre** |
| オンプレ**NetApp ONTAP**の移行・NFSとSMB両方 | **FSx for NetApp ONTAP** |
| オンプレ**ZFS**の移行 | **FSx for OpenZFS** |

> [!warning] SAAひっかけ
> - 「複数EC2で共有」だけ見てEFSを選ぶと、**Windows要件**（SMB/AD）を見落とす → EFSは**Linux専用**
> - 「高性能なファイルシステム」でEFSを選びがちだが、**HPC級のスループット要件**はLustre
> - LustreはS3「代わり」ではない。**S3と組み合わせる**高速作業レイヤー
