# S3 アクセスログ

## 概要

S3バケットへのHTTPリクエストをすべて記録する機能。誰が・いつ・どのオブジェクトに・何をしたかを、別のS3バケット（ターゲット）にログファイルとして蓄積する。

- **デフォルト: 無効**（手動で有効化が必要）

---

## 用語メモ

### バケットとは？
S3における「フォルダの最上位単位」。WindowsのCドライブやDドライブに相当するイメージ。バケットの中にオブジェクト（ファイル）を格納する。

> バケット名はグローバルで一意（世界中で重複不可）

### ベストエフォート方式とは？
「できる限り頑張るけど、保証はしない」配信方式。普通郵便のイメージ。S3アクセスログは数時間以内に配信しようとするが、遅延・欠損が起きても保証なし。

---

## 仕組み

```
[アクセス元]
     ↓ HTTPリクエスト
[ソースバケット]  →  ログを自動書き込み  →  [ターゲットバケット(ログを書き込む方)]
 (例: my-app-bucket)                        (例: my-app-logs)
```

> ソースとターゲットは**別バケット推奨**（同一にするとログが無限ループする）

---

## ログに含まれる主な情報

| 項目 | 例 |
|------|-----|
| リクエスト元 | IPアドレス、IAMユーザーARN |
| 操作の種類 | GET / PUT / DELETE |
| 対象オブジェクト | バケット名、キー名 |
| タイムスタンプ | リクエスト日時 (UTC) |
| HTTPステータス | 200 / 403 / 404 |
| 転送バイト数 | データ量 |

---

## 主なユースケース

- **セキュリティ監査**: 不審アクセスの検出
- **アクセス分析**: 利用パターン・頻度の把握
- **コンプライアンス**: 規制要件への対応・ログ保全
- **コスト分析**: どのオブジェクトが多く読まれているか

## CloudTrail との違い（試験頻出）

| 比較項目    | S3アクセスログ           | CloudTrail                  |
| ------- | ------------------ | --------------------------- |
| 記録対象    | オブジェクトへのHTTPリクエスト  | API操作（管理イベント）               |
| 例       | GET /photo.jpg     | CreateBucket / DeleteBucket |
| 粒度      | オブジェクトレベル          | バケット・アカウントレベル               |
| リアルタイム性 | 数時間の遅延あり（ベストエフォート） | ほぼリアルタイム                    |
|         |                    |                             |

## SAA 試験ポイント

1. **デフォルト無効** → 有効化が必要
2. ターゲットバケットへの書き込みには **S3 Log Delivery グループ** への権限付与が必要
3. ログ配信は**ベストエフォート**（数時間遅延、保証なし）
4. 「オブジェクト操作の記録が必要」→ S3アクセスログ
5. 「誰がバケットを作ったか知りたい」→ CloudTrail
