# CORS（Cross-Origin Resource Sharing）とは

## まず「オリジン」って何？

URLは3つの要素で構成されます。

|要素|例|
|---|---|
|スキーム|`https`|
|ホスト|`example.com`|
|ポート|`443`|

この3つが**すべて同じ**なら「同一オリジン」、1つでも違えば「別オリジン（クロスオリジン）」です。

---

## CORSとは

**リクエストを受ける側のサーバーが「どのオリジン（どのサイト）からのアクセスを許可するか」を宣言する仕組み**。
ブラウザは、別オリジンへのリクエスト時にサーバーの許可（`Access-Control-Allow-Origin` ヘッダー）を確認し、許可がなければブロックする。

> [!tip] 例え
> デパート（API側サーバー）が「許可リストに載っている店舗（オリジン）からの依頼しか受け付けない」と宣言するイメージ。**許可証を出すのは受ける側（サーバー側）**であり、ブラウザはそのルールに従うだけ。

---

## 仕組みの流れ

```
ブラウザ                      APIサーバー（別オリジン）
   |                               |
   |--- OPTIONS リクエスト -------->|  ← プリフライトリクエスト
   |   「このオリジンからアクセスOK?」 |
   |                               |
   |<-- レスポンスヘッダー ----------|
   |   Access-Control-Allow-Origin  |
   |   Access-Control-Allow-Methods |
   |                               |
   |--- 本番リクエスト（GET/POST）-->|
   |<-- レスポンス ------------------|
```

---

## AWSでCORSが関係するサービス

### ① S3（静的サイトホスティング）

フロントエンドをS3に置いて、別ドメインのAPIを叩く構成でよく発生します。

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT"],
    "AllowedOrigins": ["https://your-frontend.com"],
    "ExposeHeaders": []
  }
]
```

### ② API Gateway

Lambdaと組み合わせるときに必須。コンソールで「CORSを有効化」するか、Lambdaのレスポンスヘッダーに明示的に追加します。

```python
return {
    "statusCode": 200,
    "headers": {
        "Access-Control-Allow-Origin": "https://your-frontend.com",
        "Access-Control-Allow-Methods": "GET,POST,OPTIONS",
    },
    "body": json.dumps(result)
}
```

### ③ CloudFront

オリジンへのリクエストに `Origin` ヘッダーを転送する設定が必要です。これを忘れるとS3やAPI Gatewayの設定が正しくても機能しません。

---

## よくあるエラーと原因

|エラーメッセージ|原因|
|---|---|
|`No 'Access-Control-Allow-Origin' header`|サーバーがCORS設定していない|
|`has been blocked by CORS policy`|`AllowedOrigins`に該当オリジンがない|
|プリフライトが403|API GatewayでOPTIONSメソッドを許可していない|
|CloudFront経由だけ失敗|Originヘッダーをオリジンに転送していない|

---

## SAA試験での頻出ポイント

- CORSの設定は**リクエストを受ける側（サーバー/S3/API Gateway）に行う**
- S3のCORS設定は「バケットポリシー」ではなく**「CORSルール」として個別に設定**する
- `*`（ワイルドカード）は全オリジン許可だが、**認証情報（Cookie等）と併用不可**
- 「S3静的サイトから別ドメインのAPIを叩けない／フォントが表示されない」→ **CORS設定**を疑う

> [!info] Django開発での対応物：`django-cors-headers`。考え方は同じで、許可するオリジンをサーバー側に列挙する。
