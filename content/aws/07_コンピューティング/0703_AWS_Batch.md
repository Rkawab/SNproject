# AWS Batch

> 関連: [[0701_コンピューティングサービス一覧]]

## 概要

**大量のバッチジョブをキューに積むだけで、AWS が自動でサーバーを増減しながら並列実行してくれるフルマネージドサービス。**

---

## たとえ話：工場の仕事発注システム

> [!info] イメージをつかむ
>
> | 現実の世界 | AWS Batch |
> |---|---|
> | 工場長が仕事リストを作る | **Job Queue** に処理を登録 |
> | 「この作業はこの機械で」と設定する | **Job Definition** で処理内容を定義 |
> | 空いた作業員を自動で呼んでくる | **Compute Environment** がEC2/Fargate自動起動 |
> | 作業員が実際にこなす | **Job** がコンテナとして実行 |
>
> 「仕事をリストに積んでおくだけで、AWS が必要な分だけサーバーを自動で増やし、終わったら片付けてくれる」

---

## アーキテクチャ全体図

```
登録された処理
      ↓
[Job Queue]        ← 優先度付きで仕事を待機
      ↓ 自動振り分け
[Compute Environment]  ← EC2 / Fargate を自動増減
      ↓
[Job（コンテナ）]   ← Dockerイメージとして実行
```

---

## 主要コンポーネント

### 1. Job Definition（ジョブ定義）
- **何をするか**を定義するテンプレート
- Dockerイメージ・コマンド・CPU・メモリ・IAMロールを指定
- 一度定義すれば使い回せる（バージョン管理あり）

### 2. Job Queue（ジョブキュー）
- 実行待ちジョブを溜めておく待ち行列
- **優先度（Priority）** を設定できる（急ぎの仕事を先に処理）
- 複数の Compute Environment に紐付け可能

### 3. Compute Environment（コンピューティング環境）

| 種別 | 説明 |
|---|---|
| **Managed** | AWS が EC2/Fargate を自動管理（推奨） |
| **Unmanaged** | 自分でインスタンスを管理 |

起動タイプ（Managed の場合）:

| | EC2 | Fargate |
|---|---|---|
| サーバー管理 | 少し必要 | 完全不要 |
| コスト | スポットで大幅削減可 | やや高め |
| 向いている処理 | 大規模・長時間・GPU | 小〜中規模・短時間 |
| 起動速度 | やや遅い | 速い |
| GPUサポート | あり | なし |


## Step Functions との連携

複数のバッチジョブをワークフローで繋ぐ場合は **Step Functions + Batch** のセット構成が定番。

```
Step Functions
  ├── Job A（前処理）
  ├── Job B（メイン処理）← 並列実行も可
  ├── Job C（並列処理）
  └── Job D（後処理・集計）
```

> [!tip] SAA 頻出ポイント

### Lambda との使い分け

| 条件 | 選択 |
|---|---|
| 実行時間 **15分以内**、イベント駆動 | **Lambda** |
| 実行時間 **15分超**、大量並列、コンテナ | **AWS Batch** |

> Lambdaの15分タイムアウト制限を超えるバッチ処理 → AWS Batch が正解

### コスト最適化の定番パターン

> **EC2 スポットインスタンス + AWS Batch** の組み合わせ
> - スポットインスタンスは通常の 最大90%オフ
> - バッチ処理は中断・リトライが許容しやすいためスポット向き
> - Compute Environment でスポットの最大価格を設定するだけで自動活用

### 出題シナリオ例

**「毎晩10TBのログファイルをS3から取り出して集計したい。処理に数時間かかる。コスト最適で。」**
→ **AWS Batch + EC2スポット** が正解。Lambdaはタイムアウトで不可。

**「機械学習モデルを毎週再学習させたい。GPUが必要。」**
→ **AWS Batch + EC2（P/G系インスタンス）**。FargateはGPU非対応。

**「複雑な処理フローを順番に実行し、失敗時は特定ステップから再実行したい。」**
→ **Step Functions + AWS Batch** の組み合わせ。

---

> [!warning] 注意点
> - ジョブは **コンテナ（Docker）ベース** で動く → Dockerイメージの準備が必須
> - Fargate は **GPUサポートなし**（機械学習のGPU処理はEC2が必要）
> - 処理の **冪等性（同じ処理を何度やっても結果が変わらない設計）** を意識する（スポット中断時の再実行があるため）
