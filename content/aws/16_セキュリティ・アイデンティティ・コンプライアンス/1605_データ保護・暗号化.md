# データ保護・暗号化（金庫と鍵）

> [!info] このノートの範囲
> KMS / CloudHSM / ACM / Secrets Manager / Parameter Store。
> 「鍵」と「秘密情報」を安全に保管するサービス群。S3暗号化の文脈は [[0207_S3の暗号化]] も参照。

---

## AWS KMS（Key Management Service）

**暗号化の「鍵」を安全に作って管理するマネージド金庫。**

> [!example] たとえ
> 貴重品を守る鍵を、信頼できる銀行（AWS）が厳重に保管・貸出記録してくれる。

- S3・EBS・RDS等の暗号化キーを一元管理。**CloudTrailで鍵の利用履歴が残る**。
- マネージドHSMで保護され、AWSのほぼ全サービスと統合。

> [!tip] SAAポイント
> 「暗号化キーをAWSに管理させたい・監査ログが欲しい」→ KMS（最頻出の暗号化サービス）。
> 詳細: [[1607_AWS_KMS]]

### KMS その他の機能
- **マルチリージョンキー**：同じ鍵を別リージョンに複製。**DR（災害対策）やグローバルなS3/DynamoDBレプリケーション**で、別リージョンでも同じ暗号化が使える。
- **キーポリシー**：鍵そのものに付ける利用許可（誰がこの鍵を使えるか）。IAMポリシーと併用。
- **グラント（Grant）**：一時的・限定的に鍵の使用権を委譲する仕組み。

> [!tip] SAAポイント
> 「リージョンをまたいで暗号化データをレプリケート」→ マルチリージョンキー。

## AWS CloudHSM

**自分専用の物理鍵保管庫（シングルテナントHSM）。**

> [!example] たとえ
> 銀行の共用金庫（KMS）ではなく、**自分だけの専用金庫室**を借りる。鍵はAWSも触れない。

- **FIPS 140-2 Level 3** など、厳格な規制・コンプライアンス要件向け。

> [!tip] SAAポイント
> **KMS＝マネージド・共用／CloudHSM＝専有・自分で完全管理（厳格規制）**。「鍵を完全に自社管理」「FIPS Level 3」→ CloudHSM。

---

## AWS Certificate Manager（ACM）

**SSL/TLS証明書（https化）を無料で発行・自動更新。**

> [!example] たとえ
> サイトの「安全な接続です🔒」を証明する身分証を、自動で発行＆期限切れ前に勝手に更新してくれる。

- **ALB・CloudFront・API Gateway** に紐づけて使う。**更新忘れによる失効事故を防ぐ**のが利点。

> [!tip] SAAポイント
> 「TLS証明書の管理・自動更新」→ ACM。※EC2に直接インストールは不可、ALB/CloudFront経由が基本。

---

## AWS Secrets Manager / AWS Systems Manager Parameter Store

> [!example] たとえ
> Secrets Manager＝パスワードを定期的に自動で変えてくれる秘書。
> Parameter Store＝付箋に設定メモを貼っておく場所（鍵付き引き出し＝SecureStringも可）。

| | **Secrets Manager** | **Parameter Store** |
|---|---|---|
| 自動ローテーション | ✅ あり（RDS等と統合） | ❌ なし |
| 料金 | 有料 | **無料**（標準） |
| 主用途 | DB認証情報・APIキー・自動更新 | 設定値・シンプルな秘密 |

> [!tip] SAAポイント（頻出）
> 「**パスワードの自動ローテーションが必要**」→ Secrets Manager。
> 「無料で設定値・秘密を管理」→ Parameter Store。
> 正式には **AWS Systems Manager Parameter Store**。Systems Managerの一機能であり、単体のシークレットローテーションサービスではない。
