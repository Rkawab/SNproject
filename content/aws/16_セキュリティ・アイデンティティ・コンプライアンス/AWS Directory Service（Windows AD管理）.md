# AWS Directory Service（Windows AD管理）

## 概要
AWSでWindowsサーバー群のActive Directory（AD）を管理するサービス。オンプレミスADとの連携可否、自前運用との比較が試験の重要なポイント。

## AWS Directory Service の主要なオプション

### 1. AWS Managed Microsoft AD（最も推奨）
**特徴:**
- AWS内に本物のマネージドADを新規作成
- **パッチ管理・冗長化・バックアップは全てAWS責任**（運用最小化）
- ドメインコントローラー（DC）の冗長構成を自動構築
- オンプレADとの連携も可能（AD Trust）

**使用シーン:**
- オンプレADの管理対象が**丸ごとAWSへ移行**する場合
- 新規のAD構築が必要
- オンプレに守るべき資産が残らない
- **運用負荷を最小化したい**

**コスト:** 通常の月額課金（インスタンス料金など）

---

### 2. AD Connector（オンプレ連携）
**特徴:**
- **AWS側にAD実体を持たない**（橋渡しのみ）
- オンプレのADユーザー認証をAWS側で使用可能
- オンプレADへ依存（オンプレが停止すればAWS側も認証不可）

**使用シーン:**
- オンプレADユーザーをAWSにシームレスに統合したい
- オンプレに既存の管理対象が**残る**場合
- 既存ADを使い続けたい
- 専用線（VPN/Direct Connect）が必要

**限界:** AWS側にはADデータがないため、オンプレ障害時に単一障害点になる

---

### 3. 自前EC2でのAD運用
**特徴:**
- EC2上でActiveDirectoryを自分で構築・運用
- パッチ・バックアップ・冗長化を全て自己管理

**問題点:**
- **運用負荷が大きい**（自分で保守）
- バージョンアップ時のダウンタイム管理
- 単一EC2の場合は**単一障害点**になる

**判定:** SAAでは「運用最小化」を求める問題では不正解

---

## 試験の決め手：問題文の読み込み

### ポイント
| 条件 | 選択肢 |
|------|--------|
| **オンプレADはWebサーバー群のみ管理＋その群が丸ごとAWSへ移行** | **AWS Managed Microsoft AD**（最適） |
| オンプレに管理対象が残る＋既存ユーザーを使用 | AD Connector または Trust関係 |
| 新規AD構築＋運用最小化 | **AWS Managed Microsoft AD** |
| 自前AD構築が明記 | 自前EC2（ただし運用負荷大で避けられる傾向） |

### 「移行」問題での着眼
- **どの資産が移行するのか明確に読む**
  - Webサーバー群「のみ」→ オンプレADに管理対象が残らない
  - その場合、オンプレADと連携する理由がない → Managed AD単体で十分
  
- **「誰が何を管理するのか」を図で整理**
  - オンプレAD：実家の管理人（移行後は守る人なし）
  - Managed AD：新居に雇ったプロの管理人（AWS任せ）
  - EC2自前AD：新居で自分が管理人をやる（運用負荷大）

---

## SAA試験での出題パターン

### パターン1：最適なAD構成を選択
**問い:** Windows 2000+台がオンプレからAWSへ移行。運用を最小化したい。
**答え:** AWS Managed Microsoft AD
**理由:** オンプレ側に守る資産がなく、AWS側で自動パッチ・冗長化を受けられる

### パターン2：オンプレ連携の必要性
**問い:** 既存オンプレADを使い、AWS内のEC2も同じユーザー体系で認証したい。
**答え:** AD Connector
**理由:** オンプレに既存システムが残り、ユーザー統一が必要

### パターン3：罠選択肢
- **「オンプレADとAWS Managed ADの信頼関係を構築」** → 可能だが、本問では**不要**（オンプレに何も残らないから）
- **「各EC2でADドメイン参加」** → 当然だが、**それをどのADに参加させるか**が問題

---

## 補足：信頼関係（Trust Relationship）
- **AWS Managed AD** と **オンプレAD** の間に相互信頼を構築可能
- オンプレとAWSの両方に管理対象が**残る場合**に意味がある
- 本移行問題（オンプレ全て移行）では**不要な複雑さ**になる

---

## まとめ

| サービス | 得意分野 | 運用負荷 | 試験評価 |
|---------|---------|--------|--------|
| **AWS Managed Microsoft AD** | 新規構築＋オンプレ無し | 最小 | ⭐⭐⭐ 最優先 |
| AD Connector | オンプレ統合 | 小〜中 | ⭐⭐ 条件次第 |
| 自前EC2 AD | 完全カスタマイズ | 大 | ⭐ 避けられる |

**SAAでは「運用負荷最小」「マネージドサービス活用」が常に高評価。迷ったらManaged AD。**
