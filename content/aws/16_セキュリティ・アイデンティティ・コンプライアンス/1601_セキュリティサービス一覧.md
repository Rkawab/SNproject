# AWS セキュリティ・アイデンティティ・コンプライアンス 一覧

> [!info] このフォルダの位置づけ
> AWS認定SAAの「セキュリティ・アイデンティティ・コンプライアンス」分野をまとめたフォルダ。
> ビルのセキュリティにたとえると、各サービスは以下の役割に分かれる。

## 🏢 全体像（ビルのセキュリティのたとえ）

| 役割      | たとえ          | サービス群                                                    | ノート              |
| ------- | ------------ | -------------------------------------------------------- | ---------------- |
| 誰を入れるか  | 受付・入館証       | IAM Identity Center / Cognito / Directory Service / RAM  | [[1602_アイデンティティ系]] |
| 不審者を見張る | 監視カメラ・警備員・刑事 | GuardDuty / Inspector / Security Hub / Detective / Macie | [[1603_脅威検知・監査]]   |
| 建物の壁・門  | フェンス・ゲート・盾   | Firewall Manager / Network Firewall / Shield / WAF       | [[1604_ネットワーク防御]]  |
| 金庫と鍵    | 貴重品の保管       | KMS / CloudHSM / ACM / Secrets Manager / Parameter Store | [[1605_データ保護・暗号化]] |
| 法令順守の証明 | 監査資料         | Artifact / Audit Manager                                 | [[1606_コンプライアンス]]  |

## 🎯 SAA頻出の「対比」早見表

| 迷いどころ | 答えの軸 |
|---|---|
| GuardDuty vs Inspector | 振る舞い検知 vs 脆弱性スキャン |
| 検知→集約→調査 | GuardDuty → Security Hub → Detective |
| Macie | S3の個人情報（PII）検出 |
| Shield vs WAF | DDoS(L3/4) vs アプリ層の不正リクエスト(L7) |
| KMS vs CloudHSM | マネージド共用 vs 専有・厳格規制(FIPS L3) |
| Secrets Manager vs Parameter Store | 自動ローテーション(有料) vs 無料の設定値管理 |
| IAM Identity Center vs Cognito | 社内/複数アカウント vs アプリのエンドユーザー(B2C) |
| Artifact vs Audit Manager | AWSの証明書をもらう vs 自社の証拠を集める |
