## 第1問（Networking / CDN / LB ｜ プライベートサブネットのインターネット接続）

> [!question] 問題（再掲）
> ある企業は、複数のアベイラビリティーゾーンにまたがるVPC内で、3層構成のウェブアプリケーションを実行しています。アプリケーション層のAmazon EC2インスタンスはプライベートサブネットで稼働しており、ソフトウェアの更新プログラムをインターネットからダウンロードする必要があります。ただし、セキュリティ要件により、インターネットからこれらのEC2インスタンスへ直接アクセスできてはなりません。ソリューションアーキテクトは、アベイラビリティーゾーン障害にも耐えられる構成にする必要があります。
>
> これらの要件を満たすために実行すべきアクションの組み合わせはどれですか。（2つ選択）
>
> - A. 各AZのパブリックサブネットにNAT Gatewayを構成する。
> - B. 同一AZのNAT Gatewayへのルートを持つカスタムルートテーブルを、アプリ層のプライベートサブネットに関連付ける。
> - C. 1つのAZのパブリックサブネットにNAT Gatewayを構成し、全プライベートサブネットからそこへルーティングする。
> - D. egress-only internet gatewayへのルートを持つカスタムルートテーブルを定義し、IPv4通信にも使う。
> - E. 1つのパブリックサブネットにNATインスタンスを置き、全AZのプライベートサブネットからそこへルーティングする。

> [!success]- 答え：**A ＋ B**
> AZごとにNAT Gatewayを置き（A）、各プライベートサブネットを「同一AZ」のNAT Gatewayへルーティングする（B）ことで、外向き通信・inbound遮断・AZ障害耐性をすべて満たす。

### 要件を3つに分解する

問題文の条件をバラすと、答えが自然に決まる。

| 要件 | 言い換え | 効く部品 |
|---|---|---|
| 更新をインターネットからDLしたい | プライベートEC2の **外向き(outbound)通信** が必要 | NAT Gateway |
| 外からEC2へ直接アクセスさせない | **内向き(inbound)は遮断**、外発の戻りだけ許可 | NAT Gateway（一方通行） |
| AZ障害に耐える | **単一AZに依存しない** 冗長化 | AZごとにNAT GW ＋ 同一AZルーティング |

### たとえ話：ビルの「一方通行の宅配受取窓口」

NAT Gateway は、オフィスビルの **一方通行の宅配受取窓口** だと思うとよい。

- 社員（プライベートサブネットのEC2）は、窓口を通して **外に注文を出し（更新DL）、荷物を受け取れる**。
- でも外部の人が窓口から勝手に **社員席へ入ってくることはできない**（inbound遮断＝セキュリティ要件OK）。
- この窓口を **各フロア（AZ）に1つずつ** 置けば、1フロアが停電（AZ障害）しても、他フロアは自分のフロアの窓口で業務を続けられる（＝**A・B**）。
- もし窓口を **ビル全体で1つ** しか置かなかったら（＝**C・E**）、その窓口のあるフロアが停電すると **全フロアが外部とやり取りできなくなる**（単一障害点）。

### なぜ A と B がセットで必要か

NAT Gateway は **サブネット（＝特定のAZ）に置くAZ単位のリソース**。1台では、置いたAZが落ちると経路が切れる。だから2手が要る。

- **A：各AZのパブリックサブネットにNAT GWを置く** … 冗長化の「モノ」を用意する。
- **B：各プライベートサブネットのルートテーブルを“同じAZ”のNAT GWへ向ける** … 用意したモノを「使う経路」を引く。

A だけ（経路なし）では通信できず、B だけ（NAT GWが1台）では冗長にならない。**両方そろって初めて「外向き通信 ＋ AZ障害耐性」** が成立する。

> [!tip] なぜ「同一AZ」へ向けるのか
> AZ-aのプライベートサブネットをAZ-bのNAT GWに向けると、(1) AZ-bが落ちるとAZ-aまで巻き添えで通信不能、(2) AZをまたぐ通信で **クロスAZ転送料金** が発生。同一AZに閉じると、障害分離もコストも有利。

### 各選択肢の正誤

- **A：正解** … AZごとのNAT Gatewayで可用性を確保。マネージドなので自動スケール・（AZ内）冗長・パッチ不要。
- **B：正解** … プライベートサブネット → 同一AZ NAT GW の経路。外向きのみ許可し、戻りパケットだけ通す。
- **C：誤り** … NAT GWが1AZだけ＝単一障害点。そのAZが落ちると全AZが更新DL不能になり、「AZ障害に耐える」要件に反する。
- **D：誤り** … egress-only internet gateway（EIGW）は **IPv6専用** の外向きゲートウェイ。IPv4の更新通信には使えない。IPv4で「外向きのみ」を実現するのはNAT Gateway。
- **E：誤り** … NATインスタンスは自前運用のEC2。1台＝単一AZ依存で可用性が低く、スケール・パッチ・帯域も手動管理。NAT Gatewayの下位互換で、可用性要件に不利。

### 紛らわしい4部品の整理（SAA頻出）

| 部品 | 通信の向き | 対象プロトコル | 主な用途 |
|---|---|---|---|
| Internet Gateway (IGW) | 双方向 | IPv4 / IPv6 | パブリックサブネットの双方向インターネット接続 |
| **NAT Gateway** | **外向きのみ** | **IPv4** | **プライベートEC2の更新DL等。inboundは遮断** |
| Egress-Only IGW | 外向きのみ | **IPv6** | IPv6版のNAT相当（outboundのみ） |
| VPCエンドポイント | VPC内 → AWSサービス | － | NAT不要でS3/KMS等にプライベート接続 |

> [!note] もし行き先がAWSサービスだけなら
> 出る先がS3やDynamoDB等のAWSサービスに限られるなら、NAT Gatewayの代わりに **VPCエンドポイント** でインターネットを経由せず到達でき、NATコストを避けられる（cf. 同ファイル[[401_【問題】SAA模擬試験レベル#第8問（Networking / CDN / LB ｜ プライベートなSecrets Manager接続）|第8問]]）。ただし本問は「インターネットから更新DL」なので、汎用の外向き経路＝NAT Gatewayが必要。

### SAA的な着眼点

- 「**プライベートサブネット ＋ 外向きだけ ＋ 高可用**」が来たら、ほぼ **“AZごとのNAT Gateway”** が定番解。
- 冗長化キーワードは「**AZごとに配置**」「**同一AZへルーティング**」。1個のNAT GWは即ひっかけ。
- **EIGW＝IPv6専用** は頻出のひっかけ。IPv4なら必ずNAT。
- **NAT Gateway vs NATインスタンス**：試験では原則NAT Gateway（マネージド・高可用・自動スケール）。NATインスタンスが正解になるのは「自前で細かく制御したい」等の特殊要件のときだけ。
- コスト論点：クロスAZ課金回避のため **同一AZのNAT GW** へ向ける。
