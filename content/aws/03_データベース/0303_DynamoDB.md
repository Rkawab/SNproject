# Amazon DynamoDB

**フルマネージドのNoSQL（キー・バリュー型）データベース**。
数百万リクエスト/秒にも対応できる、スケーラビリティと低レイテンシ（1桁ミリ秒）に特化したDB。サーバー管理・容量設計が不要。

---

## 基本構造

### RDBとの対応

| RDB | DynamoDB |
|-----|----------|
| テーブル | テーブル |
| 行（Row） | アイテム（Item） |
| 列（Column） | 属性（Attribute）※アイテムごとに自由 |
| 主キー | パーティションキー（＋ソートキー） |

### キーの種類

| キー | 必須 | 役割 | 例 |
|---|---|---|---|
| **パーティションキー** | 必須 | データを**どのパーティション（物理的な置き場）に分散するか**を決めるハッシュキー | `userId`, `orderId` |
| **ソートキー** | 任意 | 同じパーティション内での**並び順**。範囲検索が可能になる | `timestamp`, `productId` |

> [!tip] 例え：図書館の分類
> パーティションキー＝「棚の番号」、ソートキー＝「棚の中での並び順」。棚番号さえ分かれば目的の本を高速で取り出せる。逆に言うと、**キー以外の属性での検索は苦手**（→ 後述のGSIで補う）。

## キャパシティモード

| | オンデマンド | プロビジョンド |
|---|---|---|
| 設定 | 不要（自動スケール） | **RCU / WCU** を事前設定 |
| 課金 | リクエスト数に応じた従量課金 | 確保した分を常時課金（安い） |
| 向き | **予測不能・スパイクのあるトラフィック** | **予測可能な安定トラフィック** |
| 備考 | — | Auto Scaling と組み合わせ可能 |

### RCU / WCU の定義

| 単位 | 内容 |
|------|------|
| 1 RCU | 強整合性読み取り 4KB×1回/秒（結果整合性なら4KB×2回/秒） |
| 1 WCU | 書き込み 1KB×1回/秒 |

---

## 整合性モデル

| | 結果整合性（デフォルト） | 強整合性 |
|---|---|---|
| 動作 | 書き込み直後は古いデータを返す可能性あり（数秒で収束） | 必ず最新データを返す |
| コスト | **0.5 RCU（半額）** | 1 RCU |
| 指定 | — | `ConsistentRead: true` |

---

## インデックス（LSI / GSI）

| | LSI（Local Secondary Index） | GSI（Global Secondary Index） |
|---|---|---|
| パーティションキー | テーブルと**同じ** | **別のキー**を指定できる |
| ソートキー | 別のキーを指定 | 別のキーを指定 |
| 作成タイミング | **テーブル作成時のみ** | **後から追加可能** |
| 整合性 | 強整合性も可能 | 結果整合性のみ |

> 「キー以外の属性で検索したい」→ **GSI** を張る、が基本パターン。

## 主要機能

| 機能 | 内容 | 典型用途 |
|---|---|---|
| **DynamoDB Streams** | テーブル変更（INSERT/MODIFY/REMOVE）をイベントストリームとして記録（保持**24時間**）。Lambdaと連携してリアルタイム処理 | 変更をトリガーに後続処理 |
| **TTL** | アイテムに有効期限（**Epoch秒**）を設定し自動削除（無料・即時ではない） | セッション・一時データ |
| **DAX** | DynamoDB専用**インメモリキャッシュ**。レイテンシを**ミリ秒→マイクロ秒**へ | 読み取り集中ワークロード |
| **トランザクション** | `TransactWriteItems` / `TransactGetItems` で**ACID**操作（最大**100アイテム**） | 残高の減算＋加算をセットで成功/失敗させる等 |
| **グローバルテーブル** | **複数リージョンへ自動レプリケーション**。マルチリージョン読み書き | DR・グローバル低レイテンシ |

```json
// TTLの例：expireAt が過ぎたら自動削除
{
  "userId": "user123",
  "sessionToken": "abc...",
  "expireAt": 1735689600
}
```

## SAA試験ポイント

| 問題文のキーワード | 答え |
|---|---|
| 「ミリ秒レイテンシ」「無制限にスケールするNoSQL」「サーバー管理不要」 | **DynamoDB** |
| 予測不能なトラフィック | **オンデマンドモード** |
| 読み取りコストを半分にしたい（多少古くてもよい） | **結果整合性読み取り（0.5 RCU）** |
| 後から別の属性で検索できるようにしたい | **GSI** |
| テーブルの変更をトリガーに処理したい | **Streams + Lambda** |
| 「マイクロ秒」の読み取りレイテンシが必要 | **DAX** |
| セッションデータを期限切れで自動削除 | **TTL** |
| 複数リージョンで読み書き・DR | **グローバルテーブル** |
