# ECS Fargate - リソース（CPU・メモリ）最適化

## 問題背景

機械学習処理などを **ECS（Fargate）** で動かしているが、各タスクに割り当てたCPUとメモリが不適切（足りない、または過剰）により、以下が発生：
- **パフォーマンスの低下**（リソース不足）
- **コストの浪費**（過度なプロビジョニング）

> [!question] **「ちょうどいいCPU・メモリの量にして、性能もコストも良くしたい」という要件に対応するサービスは？**

---

## 正解：AWS Compute Optimizer ⭐

### Compute Optimizer とは

**AWS が提供する機械学習ベースの最適化提案サービス**

- CloudWatch メトリクスから実際の使用状況を分析
- **「CPUはこれくらいが最適」「メモリはこれくらいが最適」と推奨設定を自動提案**
- EC2 / Lambda / EBS / ECS タスク など複数リソースに対応
- **無料サービス**

### ECS Fargate での活用例

```
タスク定義の現在設定
├── CPU: 1 vCPU
├── メモリ: 2 GB
└── 結果: レスポンス遅い、タイムアウト多発

  ↓ Compute Optimizer が分析

Compute Optimizer からの提案
├── CPU: 2 vCPU （推奨）
├── メモリ: 4 GB （推奨）
└── 予想削減コスト: -10% ← 最適化で実はコストも下がる場合がある
```

### 特徴

- **CloudWatch メトリクスに基づいた機械学習分析**
  - CPU利用率
  - メモリ利用率
  - ネットワーク利用率
  - など

- **右サイジング（Right-sizing）**
  - 過度にプロビジョニングされたリソース → 削減提案
  - 不足しているリソース → 増加提案

- **実装は手動**（推奨に従う/従わないは利用者の判断）
  - ワンクリック実装オプションもあり

---

## 他の選択肢との違い

### A. AWS Auto Scaling ❌

**機能**: タスク/インスタンスの **「数」を増減**

- 「このタスクを10個から15個に増やそう」という対応
- 今回の「1つのタスクのCPU/メモリが足りない」という問題は解決しない

**場面**: 負荷変動に応じて タスク数を動的にスケール

---

### B. AWS Cost Explorer ❌

**機能**: コスト使用履歴の **分析・可視化**

- 「EC2に月5万円使っている」というコスト額は分かる
- でも「最適なCPU/メモリ設定」は教えてくれない
- 自分で分析・判断する必要がある

**場面**: コスト傾向分析、費用レポート作成

---

### C. AWS Compute Optimizer ✅ **正解**

**機能**: リソース設定の **最適化提案**

- 「このECSタスクは 2vCPU / 4GB メモリが最適ですよ」と具体的に推奨

**場面**: EC2・Lambda・ECS など個別リソースの右サイジング

---

### D. Amazon CloudWatch ❌

**機能**: リソースの **監視・メトリクス表示**

- CPU使用率: 70% → グラフで見える
- メモリ使用率: 85% → グラフで見える
- **でも「では何に変更すればいいか」は自分で考える必要がある**
- 手間がかかり、最適な設定を見つけるのが難しい

**場面**: アラート設定、メトリクス監視、ダッシュボード作成

---

## 実装フロー

```
① Compute Optimizer を有効化
   ↓
② CloudWatch メトリクスの収集を待つ（数時間〜数日）
   ↓
③ Compute Optimizer が推奨設定を提示
   ↓
④ 推奨設定を新しいタスク定義に適用
   ↓
⑤ タスクを再起動（新タスク定義で稼働）
   ↓
⑥ パフォーマンス・コスト改善を確認
```

## SAA試験での出現ポイント

### 引っかけポイント：「右サイジング」問題

> [!warning] 試験では以下のように紛らわしく出題される

**パターン1**: 「コスト削減のため自動提案が欲しい」
→ Compute Optimizer ✅

**パターン2**: 「過去のコスト傾向を見たい」
→ Cost Explorer ✅

**パターン3**: 「リソース不足をアラートしたい」
→ CloudWatch ✅

**パターン4**: 「負荷に応じてタスク数を増減したい」
→ Auto Scaling ✅

## これだけ覚えれば十分

| 要件 | サービス |
|------|---------|
| **CPU・メモリを"ちょうどいい量"にしたい** | **Compute Optimizer** ⭐ |
| タスク数を増減したい | Auto Scaling |
| コスト額を分析したい | Cost Explorer |
| リソースを監視・アラートしたい | CloudWatch |
