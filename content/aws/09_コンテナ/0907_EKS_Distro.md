# EKS Distro（EKS-D）

> [!info] 一言でいうと
> Amazon EKS が内部で使っている Kubernetes ディストリビューションを、オープンソースとして公開したもの。
> AWS と同じ「素材」を使って、自分の好きな場所（オンプレ・他クラウド・エッジ）に Kubernetes クラスタを構築できる。

## たとえ話：人気レストランの3つの楽しみ方

人気ラーメン店「EKS亭」を例にすると分かりやすい。

| サービス | たとえ | 誰が何をやる？ |
|---|---|---|
| **Amazon EKS** | お店で食べる | 店員（AWS）が全部やってくれる |
| **[[EKS_Anywhere]]** | 出張シェフを家に呼ぶ | AWSが自宅まで来て店の味を再現 |
| **EKS Distro** | レシピと素材だけもらう | 素材は本物。調理・後片付けは全部自分 |
| **EC2に自分でK8s** | スーパーで適当に材料買って自作 | 味の保証なし。全部自己責任 |

EKS Distro は **「AWSが品質保証してくれた Kubernetes の素材一式」**。それ以外（構築・運用）は全部自分で行う。

## 中身に含まれるもの

AWS が EKS 用に **ビルド・テスト・パッチ適用** したコンポーネント一式。

- [[0908_Kubernetes]] 本体（kube-apiserver, kube-controller-manager, kube-scheduler, kubelet 等）
- etcd（クラスタ状態保存DB）
- CoreDNS（クラスタ内DNS）
- CNI プラグイン（Pod間ネットワーク）
- CSI ドライバ（ストレージ連携）
- kube-proxy

> [!tip] EKSと完全に同じバイナリ
> EKS本番で動いているのと**同じバージョン・同じパッチ**が使える。脆弱性対応も EKS と同じタイミング。

## どこで動かせる？

どこでも。

- オンプレミスのサーバー
- EC2 上に自分で構築
- 他社クラウド（Azure / GCP）
- ラズパイクラスタなどのエッジ環境

## EKS マネージドとの違い

| 項目 | Amazon EKS（マネージド） | EKS Distro |
|---|---|---|
| コントロールプレーン | AWSが管理 | **自分で構築・運用** |
| ワーカーノード | EC2/Fargate で簡単 | **自分で用意** |
| アップグレード | ボタン一発 | **自分で対応** |
| 料金 | クラスタ $0.10/h + ノード代 | **無料**（インフラ代のみ） |
| サポート | AWSサポート対象 | コミュニティのみ |
| 入手先 | AWSコンソール | [GitHub](https://github.com/aws/eks-distro) |

## コンテナサービスの責任共有スペクトラム（SAA頻出視点）

```
[フルマネージド] ←―――――――――――――→ [フルセルフ管理]

 Fargate < EKS/ECS < EKS Anywhere < EKS Distro < 自前K8s
   ↑                                              ↑
   AWSが面倒を見る範囲が広い        自分で面倒を見る範囲が広い
```

## いつ EKS Distro を選ぶ？

- ✅ オンプレや他クラウドで「EKS と同じ品質の K8s」が欲しい
- ✅ AWSサポート契約は不要、コスト最小にしたい
- ✅ 既存の運用チームが K8s に十分習熟している
- ❌ AWSサポートが必要 → [[EKS_Anywhere]] を選ぶ
- ❌ AWS上で運用するだけ → 素直に [[0906_EKS]] を選ぶ

## SAA試験ポイント

問題文のキーワードと選ぶサービスの対応：

| 問題文のキーワード | 答え |
|---|---|
| 「オンプレで AWS と同じ K8s、AWSサポート付き」 | EKS Anywhere |
| 「オンプレで EKS と同じ K8s、無料・サポート不要」 | **EKS Distro** |
| 「AWS上でフルマネージドK8s」 | EKS |
| 「サーバー管理したくない、コンテナだけ動かしたい」 | Fargate（[[0904_ECS]] or EKS on Fargate） |

EKS Distro 単独より、**EKS シリーズ全体の比較**で覚えるのが効率的。
