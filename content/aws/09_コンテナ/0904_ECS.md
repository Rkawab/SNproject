# Amazon ECS（Elastic Container Service）

Amazon ECSとは、Dockerコンテナを管理・実行するための**フルマネージドコンテナオーケストレーションサービス**。コンテナの起動・停止・スケーリング・配置をAWSが自動管理してくれる。

---

## たとえ話：デリバリー弁当センター

| 概念 | 弁当センターのたとえ |
|------|---------------------|
| コンテナ | お弁当（アプリそのもの） |
| タスク | 注文1件（1つ以上のコンテナセット） |
| サービス | 「常に10個の弁当を維持し続ける」契約 |
| クラスター | デリバリーセンター全体 |
| Fargate | 完全外注の業者（車も人もセンターが用意） |
| EC2起動タイプ | 自分でバイクを買って配達員を雇う |

---

## ECSクラスターとは

**クラスター = コンテナを動かすための「論理的な作業場所（グループ）」**

- タスク・サービスをまとめて管理する単位
- 実際のコンピューティングリソース（EC2 or Fargate）をひとくくりにする
- 1つのクラスター内に複数のサービスを共存させることができる

```
ECSクラスター（例: my-app-cluster）
├── サービスA（Web API）  → タスク × 3個 稼働中
├── サービスB（バッチ処理）→ タスク × 1個 稼働中
└── サービスC（ワーカー） → タスク × 2個 稼働中
```

## 起動タイプの比較

| 項目 | Fargate | EC2起動タイプ |
|------|---------|--------------|
| インフラ管理 | AWSが全て管理 | 自分でEC2を管理 |
| コスト | 使った分だけ（vCPU・メモリ秒単位） | EC2を常時起動 |
| カスタマイズ性 | 低め | 高い（GPUインスタンス等も可） |
| 運用負荷 | ほぼゼロ | OSパッチ・スケール管理が必要 |
| 適したケース | 標準的なWebアプリ・API | GPU処理・特殊OS設定が必要な場合 |

> [!tip] Fargateは「コンピューティングリソースを指定せずにコンテナを実行できる」という点がポイント。

## コントロールプレーンを気にしなくていい理由

Kubernetesを自前で動かす場合、「コントロールプレーン（マスターノード）」の管理が必要。  
ECSでは**コントロールプレーンは常にAWSが完全管理**している。

| サービス | コントロールプレーン | ノード（サーバー） |
|---------|---------------------|--------------------|
| ECS + Fargate | AWSが管理 | AWSが管理 |
| ECS + EC2 | AWSが管理 | **自分が管理** |
| EKS + Fargate | AWSが管理 | AWSが管理 |
| EC2に自前K8s | **自分が管理** | **自分が管理** |

Fargateを選べばコントロールプレーンもノードも両方AWSが面倒を見てくれる。  
「インフラ面を気にしなくていい」とはこの状態を指す。

## 主要コンポーネント

| 用語 | 説明 |
|------|------|
| **タスク定義** | コンテナの設計図（Dockerイメージ・CPU・メモリ・環境変数・ポート） |
| **タスク** | タスク定義を元に実際に起動したコンテナの実体 |
| **サービス** | タスクの数を維持・管理する仕組み（ELBとの連携も担う） |
| **クラスター** | タスク・サービスをまとめる論理グループ |

> [!info] タスク定義はDockerのdocker-compose.ymlに近いイメージ。サービスはその「常時稼働保証」を担う。

## SAA試験 頻出ポイント

> [!warning] 試験ではシナリオから「ECS + Fargate か EKS か」を判断させる問題が多い。

### よく出る選択基準

| シナリオ | 正解 |
|---------|------|
| サーバーレスでコンテナを動かしたい | ECS + Fargate |
| Kubernetes互換が必要 | EKS |
| GPUが必要なコンテナ | ECS + EC2（FargateはGPU非対応） |
| マイクロサービス間のサービス検出 | AWS Cloud Map |
| コンテナのログ管理 | CloudWatch Logs |
| Blue/Greenデプロイ | CodeDeploy + ECS |

### よく出る設問パターン

- 「最小限の運用負荷でコンテナを実行するには？」→ **ECS + Fargate**
- 「EC2の管理なしにコンテナをスケールするには？」→ **ECS + Fargate**
- 「既存のKubernetesワークロードをAWSに移行するには？」→ **EKS**
