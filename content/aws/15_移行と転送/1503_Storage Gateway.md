# AWS Storage Gateway

> [!info] このノートについて
> オンプレミスとAWSストレージを橋渡しする **ハイブリッドクラウドストレージ** サービス。SAAでは「ローカルの低レイテンシ + クラウドの耐久性」を両立する選択肢として頻出。**4タイプの使い分け**（File / FSx File / Volume / Tape）が問われる。

## ひとことで言うと

オンプレに置く小さな「**受付窓口＋手元キャッシュ棚**」。よく使う荷物だけ手元（ローカルキャッシュ）に置き、本体はAWS（S3 / Glacier / EBSスナップショット / FSx）の巨大倉庫に保管する。アプリ側は今までどおり **NFS / SMB / iSCSI / VTL** で繋ぐだけでよい。

> [!example] 例え：街中の宅配ロッカー
> 倉庫（AWS）は郊外にあるが、駅前のロッカー（オンプレのゲートウェイ）に「よく取り出す荷物」をキャッシュしておく。利用者は駅前で素早く受け取れ、実体はクラウドに安全保管。回線越しでも体感が速い。

---

## 4つのゲートウェイタイプ

| タイプ | オンプレ側のインターフェース | バックエンド保存先 | 主な用途 |
|---|---|---|---|
| **S3 File Gateway** | NFS / SMB（ファイル共有） | **S3**（オブジェクト） | ファイルサーバーの実体をS3へ。よく使う分はローカルキャッシュ |
| **FSx File Gateway** | SMB | **FSx for Windows File Server** | オンプレからFSxへ低レイテンシでアクセス（キャッシュ経由） |
| **Volume Gateway** | iSCSI（ブロック） | **S3 + EBSスナップショット** | ブロックボリュームをクラウド連携。下記2モードあり |
| **Tape Gateway (VTL)** | iSCSI VTL（仮想テープ） | **S3 → Glacier / Deep Archive** | 物理テープ運用をそのままクラウドへ |

### Volume Gateway の2モード（試験頻出の引っかけ）

| モード | データの本体 | クラウド側 | 特徴 |
|---|---|---|---|
| **キャッシュ型（Cached）** | **S3（クラウド）** が本体 | S3 | ローカルは「よく使う分」だけキャッシュ。容量をクラウドに寄せたい |
| **保管型（Stored）** | **オンプレ** が本体（全データ） | S3へ非同期バックアップ（EBSスナップショット） | 低レイテンシ最優先。クラウドはバックアップ用 |

> [!tip] キャッシュ型 vs 保管型 の覚え方
> - 「**全データをローカルに置きたい**」（低レイテンシ最優先・クラウドはバックアップ）→ **保管型(Stored)**
> - 「**ローカル容量を節約したい**」（本体はクラウド・手元はキャッシュ）→ **キャッシュ型(Cached)**

## SAA試験での判断ポイント

| 問題文のキーワード | 正解タイプ |
|---|---|
| NFS/SMBのファイル共有をS3バックエンドで提供したい | **S3 File Gateway** |
| 物理テープ／VTL／既存バックアップソフトをそのままクラウドへ | **Tape Gateway** |
| iSCSIのブロックボリューム＋クラウドバックアップ | **Volume Gateway** |
| 全データはオンプレ保持、S3へバックアップだけしたい | **Volume Gateway（保管型）** |
| ローカル容量を抑えつつブロックストレージを使いたい | **Volume Gateway（キャッシュ型）** |
| オンプレからFSx for Windowsへ低レイテンシアクセス | **FSx File Gateway** |

> [!important] 共通の決め手
> 「**オンプレを残したまま（ハイブリッド）**」「**既存のプロトコル（NFS/SMB/iSCSI/VTL）を変えずに**」「**ローカルキャッシュで低レイテンシ**」——この3つが揃ったら Storage Gateway。

## 紛らわしいサービスとの違い

| 比較対象 | Storage Gateway | 相手 | 判断軸 |
|---|---|---|---|
| **DataSync** | 継続的な**ハイブリッドアクセス**（キャッシュ経由で常用） | 一括・定期の**ファイル転送/移行** | 「使い続ける」=SG / 「移して終わり・定期同期」=DataSync |
| **Snow Family** | **オンライン**でクラウドを常用 | **オフライン**物理輸送（回線が細い/超大容量） | 回線で間に合う=SG / 回線で数週間=Snow |
| **Direct Connect** | ストレージのハイブリッド接続 | 専用線そのもの（NW回線） | SGは“窓口”、DXは“道路” |

> [!warning] DataSyncとの線引き（混同注意）
> 「SMB→FSx への**定期バックアップ／同期／転送**」が目的なら → [[1502_DataSync]]。Storage Gateway が主役になるのは **オンプレを残してキャッシュ経由でクラウドを使い続ける（常用のハイブリッドアクセス）** が要件のとき。「移して終わり・定期同期」か「使い続ける」かで切り分ける。

## まとめ早見表

> [!important] Storage Gateway タイプ選択フロー
> ```
> オンプレを残してAWSストレージと繋ぎたい
> │
> ├─ ファイル共有(NFS/SMB)で…
> │   ├─ 実体をS3へ              → S3 File Gateway
> │   └─ 実体をFSx for Windowsへ → FSx File Gateway
> ├─ ブロック(iSCSI)で…
> │   ├─ 本体はクラウド(容量節約) → Volume Gateway（キャッシュ型）
> │   └─ 本体はオンプレ(低遅延)   → Volume Gateway（保管型）
> └─ テープ運用(VTL)をそのまま   → Tape Gateway（→Glacierへアーカイブ）
> ```
