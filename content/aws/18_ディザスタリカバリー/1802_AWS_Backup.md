# AWS Backup｜バックアップの一元管理

> [!info] このノートの位置づけ
> [[1801_DR戦略4パターン（RTO_RPO）]] の「バックアップ&リストア」戦略で登場する **AWS Backup** の専用ノート。模試では「**複数サービスのバックアップを一元管理**」「**ランサムウェア対策**」の文脈で出る。

## 何者か

**AWSの各サービスのバックアップを、1か所のルールでまとめて自動化・管理するサービス**。

各サービスには個別のバックアップ機能（EBSスナップショット、RDS自動バックアップ…）があるが、サービスごとにバラバラに設定・管理するのは大変。AWS Backupは、その上に立つ**司令塔**。

> [!tip] たとえ話：ビル全体の防災管理会社
> 各フロア（EBS・RDS・EFS…）が自分で消火器を管理する（＝個別スナップショット）代わりに、**防災管理会社（AWS Backup）と1本契約**すると、「毎晩点検・35日保管・遠隔地に控えも保管」というルールを**ビル全体に一括適用**してくれる。

---

## 中核の3部品

| 部品 | 役割 |
|---|---|
| **バックアッププラン** | 「いつ・どの頻度で・何日保管するか」のルール（例：日次・35日保持・コールドへ移行） |
| **リソース割り当て** | プランを適用する対象（**タグで一括指定**できるのが強み。例：`env=prod` に日次バックアップ） |
| **バックアップボールト** | バックアップの保管庫。KMSで暗号化 |

### 対応サービス（主なもの）

EC2 / **EBS** / **RDS**・Aurora / **DynamoDB** / **EFS** / **FSx** / Storage Gateway / **S3** / DocumentDB / Neptune など横断的に対応。

---

## SAA頻出ポイント

### ① 一元管理・自動化

> 「EBS・RDS・EFSのバックアップを**一元的に管理し、保持ポリシーを統一**したい。**運用負荷は最小限**に」

→ 個別にスクリプト・Lambda・EventBridgeでスナップショットを取る選択肢は**運用負荷が大きい誤答**。**AWS Backupのバックアッププラン**が正解。

### ② クロスリージョン／クロスアカウントコピー

- バックアップを**別リージョン**へ自動コピー → DR（[[1801_DR戦略4パターン（RTO_RPO）]]のバックアップ&リストア戦略）
- **別アカウント**へ自動コピー → 本番アカウント侵害時でもバックアップが無事

### ③ Backup Vault Lock（ランサムウェア対策）

ボールトを **WORM**（Write Once Read Many＝一度書いたら変更・削除不可）にできる。

> 「**ランサムウェア対策**として、バックアップが**管理者でも削除・改変できない**ようにしたい」

→ **Backup Vault Lock**。S3の同種機能は **S3 Object Lock**（→ [[0205_S3のバージョン管理]]）。

| 守る対象 | 機能 |
|---|---|
| バックアップ全般 | **AWS Backup Vault Lock** |
| S3オブジェクト | **S3 Object Lock** |

## 紛らわしいサービスとの整理

| サービス | 役割 | AWS Backupとの違い |
|---|---|---|
| **EBSスナップショット / RDS自動バックアップ** | 各サービス個別のバックアップ | 単一サービスならこれでも可。**複数サービス横断・統一ポリシー**ならAWS Backup |
| **Amazon Data Lifecycle Manager（DLM）** | EBSスナップショット**専用**の自動化 | EBSだけ・追加コストなしで済むならDLM。横断管理はAWS Backup |
| **AWS Elastic Disaster Recovery（DRS）** | サーバー丸ごとの**継続レプリケーション**（低RTO/RPO） | Backupは「時点コピー」、DRSは「常時複製して数分で起動」。RTO要件が厳しければDRS |
| **S3バージョニング / レプリケーション** | S3内の世代管理・複製 | S3単体の保護。横断ポリシーはAWS Backup |

> [!warning] SAAひっかけ
> - 「バックアップの一元管理」問題で **Lambda＋EventBridgeで自作**する選択肢 → 動くが**運用負荷最小の要件に反する**誤答
> - 「RTO数分」の要件にAWS Backup（リストアに時間がかかる）→ **DRS / ウォームスタンバイ**が正解
> - 「削除できないバックアップ」＝ Vault Lock。**バックアップの暗号化（KMS）とは別の話**
