# AWS コスト管理サービス一覧

## 概要
AWS のコスト最適化・管理に関連する主要サービス。SAA試験では **3つのサービスの違い（最適化提案／可視化・分析／予算アラート）** が引っかけポイントになりやすい。

---

## 1. Compute Optimizer ⭐

### 機能
- **AWS が提供する機械学習ベースの最適化提案サービス**
- EC2インスタンス、Lambda関数、EBSボリュームなどのリソースの**適正サイズを自動提案**
- 料金最適化だけでなく、パフォーマンス最適化も同時に行う

### 特徴
- **無料サービス**
- CloudWatch のメトリクスに基づいて分析
- 提案内容：CPU利用率、メモリ利用率、ネットワーク利用率などから判定
- **ライトサイジング（右サイジング）** = 過度にプロビジョニングされたリソースを削減

### SAA試験での出現ポイント
- **「コスト削減のための自動提案」→ Compute Optimizer**
- 最適化内容は提案に留まり、実装は手動（ワンクリック実装オプションあり）

### 注記
既に [[1401_マネジメント・ガバナンスサービス一覧]] の「⑤ 最適化・コンサル系」に詳細記載あり

---

## 2. AWS Cost Explorer ⭐⭐

### 機能
- AWS の使用料金を **可視化・分析するためのツール**
- 時系列グラフ、カテゴリ別（サービス別、タグ別、リージョン別）の分析が可能
- 過去の使用状況から将来のコスト予測も可能

### 特徴
- **ビジュアルデータ分析に特化**
- EC2 Reserved Instance（RI）や Savings Plans の購入シミュレーション機能
- 過去12ヶ月のデータ確認可能
- **「どのサービス・どのリージョンでコストが掛かっているか」を把握する**

### SAA試験での出現ポイント
- **「コスト分析・可視化」→ Cost Explorer**
- Budgets との違い：Explorer は分析・予測、Budgets は予算管理
- 選択肢では **AWS Cost Explorer** と正式名で出ることがある

### 使用シーン
- 経営層へのコスト報告時に、わかりやすいグラフを提示したい
- サービス別・リージョン別のコスト分析
- RI/Savings Plans の購入判断

---

## 3. AWS Budgets ⭐⭐

### 機能
- AWS 支出に対する **予算を設定・管理するサービス**
- 予算上限に達した場合、**アラート通知**を自動で発送
- SNS 通知、メール通知に対応

### 特徴
- **予算管理とアラート機能に特化**
- カスタム予算の設定（サービス別、タグ別、Cost Category 別）
- 実績値と予測値の両方でアラート発火可能
- 予算超過時の **自動対応ルール（IAM ポリシー連携で即座にリソース削除など）** は別途設定

### SAA試験での出現ポイント
- **「予算超過時にアラートを出したい」→ Budgets**
- Cost Explorer との違い：分析ではなく、**予算制御・コンプライアンス**

### 使用シーン
- 部門別・プロジェクト別に予算を割り当て、超過を防止
- チーム開発時にコスト暴走の早期検知

---

## 4. Cost and Usage Report (CUR)

### 機能
- AWS の詳細な使用料金データを **CSV形式で S3 に定期的に出力**
- グラニュラリティ（詳細度）が最も高い料金データ取得方法

### 特徴
- サービス別・リソース別・タグ別など最細粒度のデータ取得可能
- 外部ツール（Athena, QuickSight, BI ツール）で深い分析が可能
- **手動設定が必要**（Console → Billing and Cost Management）
- 生成タイミング：1日1回、深夜（UTC）

### SAA試験での出現ポイント
- **「S3 に自動出力される詳細レポート」→ Cost and Usage Report**
- 分析用途（Excel/BI連携）

### 使用シーン
- より詳細な内部分析、カスタムレポート作成
- 部外者監査やコンプライアンスレポート

---

## 5. Trusted Advisor ⭐（コスト観点のみ）

### 機能
- AWS インフラの **ベストプラクティス検査** サービス
- セキュリティ、パフォーマンス、信頼性、コスト最適化 の4軸で検査

### コスト最適化観点
- **未使用の EIP（Elastic IP）検出**
- **アイドル状態の RDS インスタンス検出**
- **低利用率の EC2 インスタンス検出** （Compute Optimizer と被る部分あり）
- **未使用の CloudWatch Alarms** など

### 特徴
- **ビジネスサポート**以上で全項目の詳細チェック可能
- フリープランでは限定的（セキュリティグループ、EBS スナップショット程度）

### SAA試験での出現ポイント
- **「未使用リソース検出」→ Trusted Advisor**
- Compute Optimizer との違い：Optimizer は「最適化提案」、Advisor は「未使用検出」

---

## 6. コスト配分タグ（Cost Allocation Tags）

### 機能
- リソースに付けたタグ（`部門=営業` `プロジェクト=X` など）を**コストの集計軸**として使えるようにする仕組み
- **Billingコンソールでタグを「コスト配分タグ」として有効化**して初めて Cost Explorer / CUR で集計に使える（**タグを付けただけでは集計されない**のが引っかけ）
- 2種類：**ユーザー定義タグ**（自分で付ける）と **AWS生成タグ**（`aws:createdBy` など）

### SAA試験での出現ポイント
- 「**部門ごと・プロジェクトごとにコストを配賦（チャージバック）したい**」→ コスト配分タグ＋Cost Explorer
- **有効化した時点以降**のデータから集計される（過去に遡っては適用されない）

---

## 🎯 SAA試験での出現パターン

### パターン1：「コスト削減の自動提案が欲しい」
→ **Compute Optimizer**（EC2/Lambda の右サイジング）

### パターン2：「過去のコスト傾向を分析したい」
→ **Cost Explorer**（時系列分析、サービス別分析）

### パターン3：「予算超過時に通知したい」
→ **AWS Budgets**（アラート設定、SNS 連携）

### パターン4：「詳細なコストデータを外部ツール連携したい」
→ **Cost and Usage Report**（S3 出力、Athena/QuickSight 連携）

### パターン5：「未使用リソースを発見したい」
→ **Trusted Advisor**（EIP、アイドル RDS など）

### パターン6：「部門ごとにコストを配賦したい」
→ **コスト配分タグ**（Billingで有効化 ＋ Cost Explorer で集計）

## 補足：似ているサービス比較表

| サービス | 主な機能 | 提案/アラート | 詳細度 |
|---|---|---|---|
| **Compute Optimizer** | 最適化提案 | 📊 提案のみ | 中（サービス・リソース別） |
| **Cost Explorer** | 可視化・分析 | ❌ なし | 中（時系列・カテゴリ別） |
| **AWS Budgets** | 予算管理 | 🔔 アラート | 低〜中（予算設定単位） |
| **Cost and Usage Report** | 詳細データ出力 | ❌ なし | 高（最細粒度） |
| **Trusted Advisor** | ベストプラクティス検査 | 📋 検査結果報告 | 中（リソース別） |

## 学習時の留意点

1. **Compute Optimizer と Trusted Advisor の使い分け**
   - Optimizer：「このEC2は m5.large が最適」と**具体的なサイズ提案**
   - Advisor：「このEIPは未使用」と**リソース状態の報告**

2. **Cost Explorer と Budgets の使い分け**
   - Explorer：「過去30日、EC2に月3万円」→ **分析・レポーティング**
   - Budgets：「EC2予算5万円に設定、4万5千円で警告」→ **制御・ガバナンス**

3. **Cost and Usage Report が選ばれるケース**
   - S3、Athena、QuickSight などの「外部ツール連携」が明記されている場合
