# AWS Data Pipeline

> [!warning] 旧世代サービス。今は [[1107_AWS Glue]] などへの置き換えが推奨。「昔こういうのがあった」程度でOK。
> 用語が不安なら先に [[1102_データ分析の超基礎・用語集]] を。担当は分析5段階のうち**①収集**（定期コピー）。

## 超かんたんに言うと

「毎晩このデータをここからあそこへコピーする」といった、**定期的なデータの移動・変換を予約実行する**ための古いサービスです。

ただし**今から新しく使うのは非推奨**。AWS自身が新規開発を止めていて、後継サービスに役割を譲っています。

> [!example] たとえ話
> 昔の宅配便の「集荷スケジュール表」。今はもっと賢い自動システム（[[1107_AWS Glue]] など）に置き換わりました。

## 後継サービス（今ならこれを使う）

| やりたいこと | 今の正解 |
|---|---|
| データの整形・変換（[[1102_データ分析の超基礎・用語集|ETL]]） | [[1107_AWS Glue]] |
| 複数処理の順番を組む（ワークフロー） | AWS Step Functions |
| Airflow という有名ツールで組みたい | Amazon MWAA |

## 試験での注意

- 出題されることはありますが、近年は **[[1107_AWS Glue]]** や **Step Functions** が正解になることが多いです。
- 「マネージドのデータ変換ワークフロー」と聞いても、**反射的に Data Pipeline を選ばない**こと。まず [[1107_AWS Glue]] を疑いましょう。
