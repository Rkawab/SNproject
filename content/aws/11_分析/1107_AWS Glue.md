# AWS Glue

> [!info] バラバラなデータを「使いやすい形に整える下ごしらえ係」。サーバー管理は不要。
> 用語が不安なら先に [[1102_データ分析の超基礎・用語集]] を。担当は分析5段階のうち**③加工**。

## 超かんたんに言うと

集めてきたデータは、形がバラバラだったり、余計な項目が入っていたりして、そのままでは分析しにくいものです。
Glue は、その **「データの下ごしらえ（整形・変換・移し替え）」を自動でやってくれる係**です。この下ごしらえ作業を専門用語で [[1102_データ分析の超基礎・用語集|ETL]] と呼びます。

しかも**サーバーを自分で用意しなくてよい**（[[1102_データ分析の超基礎・用語集|サーバーレス]]）ので、使いたいときに動かして、使った分だけ払えばOK。

> [!example] たとえ話
> Glue は「**自動野菜カッター**」。泥つきの野菜（バラバラのデータ）を入れると、洗って・切って・きれいに並べて（整形して）別の棚（[[0201_S3]] や分析用DB）に出してくれます。

## Glue の3つの顔

Glue は1つの名前で3つの役割を持っています。ここが分かりにくいので分けて覚えましょう。

### ① Glue ETL（下ごしらえ作業そのもの）
実際にデータを整形・変換する処理。画面上でマウス操作（Glue Studio という GUI）で組み立てることもでき、プログラムを自動生成してくれます。

### ② Glue Data Catalog（データの説明書＝目録）
「このファイルは売上表で、1列目が日付…」という**[[1102_データ分析の超基礎・用語集|データカタログ]]（説明書）**を管理する図書館の目録。
ここが重要で、[[1109_Amazon Athena]] や [[1108_Amazon EMR]] など他のサービスも**この同じ目録を見て**データを扱います。みんなで共有する「データの索引」です。

### ③ Glue Crawler（自動で説明書を作る小人）
[[0201_S3]] の中を定期的に見回って、「どんなファイルがあって、どんな列構成か」を**勝手に調べて目録（②）に登録**してくれる仕組み。
> [!example] 冷蔵庫の中身を勝手にチェックして、ラベルを貼ってくれる小人さん。

## こんなときに使う（試験頻出シナリオ）

| やりたいこと | 答え |
|---|---|
| サーバー管理なしでデータの整形（ETL）をしたい | **Glue** |
| [[0201_S3]] のデータの「説明書（カタログ）」を管理したい | **Glue Data Catalog** |
| 新しいファイルの構成を自動で読み取ってほしい | **Glue Crawler** |
| [[1109_Amazon Athena]] と組み合わせてサーバーレス分析したい | **Glue + Athena**（超頻出の鉄板コンビ） |

## Glue と [[1108_Amazon EMR]] の使い分け

どちらも「データの加工」担当ですが、規模が違います。
- **Glue**：手軽・管理不要。小〜中規模の下ごしらえ向け。
- **[[1108_Amazon EMR]]**：超大量データを本格的に処理する大厨房。設定の自由度が高いぶん手間もかかる。
