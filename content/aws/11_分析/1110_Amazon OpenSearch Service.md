# Amazon OpenSearch Service

> [!info] 大量のログや文章から、キーワードで一瞬で探し出す「検索エンジン」のマネージド版。
> 用語が不安なら先に [[1102_データ分析の超基礎・用語集]] を。担当は分析5段階のうち**④分析**（検索）。
> 旧名は Amazon Elasticsearch Service。

## 超かんたんに言うと

サーバーやアプリは毎日、膨大な量の**ログ（動作の記録）**を吐き出します。
その山のような記録の中から、**「エラーを含む、直近1時間の、特定の人のアクセス」だけを一瞬で見つけ出す**――それを可能にするのが OpenSearch です。Googleの検索（[[1102_データ分析の超基礎・用語集|全文検索]]）を、自分のデータに対してできるイメージ。

> [!example] たとえ話
> 図書館の本（ログ）が床いっぱいに山積み。普通なら1冊ずつめくって探します。
> OpenSearch は**あらかじめ全ページに索引を付けておく**ので、「この言葉が載っているページ」を即座に出せる「社内版Google」です。

## どんなことに使う？

| 用途 | 説明 |
|---|---|
| **ログ分析** | サーバーやアプリの記録をリアルタイムに検索・調査 |
| **アプリ内検索** | ネットショップの「商品検索」窓、社内文書の検索 |
| **監視・アラート** | 数値の異常を見つけて知らせる |
| **セキュリティ調査** | 不審なアクセスを記録から相関分析 |
| **ベクトル検索** | 生成AI（RAG）の「意味が近い文書を探す」用途にも対応 |

## 付属の可視化ツール

OpenSearch には **OpenSearch Dashboards** という、検索結果をグラフで見せる画面が付いています（旧名 Kibana）。検索した結果をそのまま可視化できます。
管理不要の **OpenSearch Serverless** 版もあります。

## 定番の流れ

```
アプリのログ → Kinesis Firehose → OpenSearch → Dashboards で見る
（記録が出る   集めて運ぶ          検索できる形に  グラフ化）
```
（Kinesis については [[1103_Amazon Kinesis]] を参照）

## こんなときに使う（試験頻出シナリオ）

| やりたいこと | 答え |

## 似たサービスとの違い（ここ混同注意）

| | OpenSearch | [[1109_Amazon Athena]] | [[1111_Amazon QuickSight]] |
|---|---|---|---|
| 役割 | 検索・即時調査 | 手軽なSQL分析 | きれいなダッシュボード作り |
| 速さ | ミリ秒で返る | 数秒〜 | 数秒〜 |
| 向いている場面 | ログ調査・検索窓 | S3データの単発分析 | 経営報告・可視化 |
