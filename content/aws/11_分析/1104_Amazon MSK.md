# Amazon MSK (Managed Streaming for Apache Kafka)

> [!info] 世界標準のデータ配送ソフト「Apache Kafka」を、AWSが代わりに運用してくれるサービス。
> 用語が不安なら先に [[1102_データ分析の超基礎・用語集]] を。担当は分析5段階のうち**①収集**（リアルタイム）。

## 超かんたんに言うと

**Apache Kafka（カフカ）** という、リアルタイムにデータを流すための**世界中で使われているオープンソースのソフト**があります。
このKafkaは高機能ですが、自分でサーバーを立てて運用するのは手間がかかります。MSK は、その**Kafkaの運用（サーバー管理・故障対応など）をAWSが肩代わり**してくれるサービスです。Kafka本来の使い方・ツールはそのまま使えます。

> [!example] たとえ話
> [[1103_Amazon Kinesis|Kinesis]] が「**AWS純正の配送便**」だとすると、
> MSK は「**世界標準の宅配業者（Kafka）を、AWSが代わりに運営**してくれる」イメージ。
> すでにKafkaという宅配業者と取引（＝既存システム）があるなら、業者を変えずにそのまま使える、というのが利点です。

## なぜ Kinesis ではなく MSK を選ぶ？

| 観点 | MSK | [[1103_Amazon Kinesis]] |
|---|---|---|
| 中身 | Apache Kafka（世界標準のOSS） | AWS独自の仕組み |
| 既存ツール | Kafka用ツールがそのまま使える | AWSサービスとの連携が得意 |
| 引っ越しやすさ | 他社クラウド・自社サーバーとも互換 | AWS前提になりがち |
| 管理の手間 | やや必要 | ほぼ不要 |

ひとことで言うと、**「すでにKafkaを使っている／Kafkaの仕組みを使いたい」なら MSK**、**「AWSで一から手軽に始めたい」なら Kinesis**。

## 3つの提供形態

| 形態 | 特徴 |
|---|---|
| **MSK Provisioned**（標準） | サーバーの台数・性能を自分で指定する |
| **MSK Serverless** | サーバー管理不要。使った分だけ・自動で増減 |
| **MSK Connect** | DBやファイルとKafkaを**つなぐ配線**を簡単に用意できる |

## こんなときに使う（試験頻出シナリオ）

| やりたいこと | 答え |
|---|---|
| すでに動いている Kafka を AWS へ移したい | **MSK** |
| Kafka専用のツール（Connect等）を使いたい | **MSK** |
| 他社クラウドや自社サーバーとも互換性が欲しい | **MSK** |
| AWS だけで手軽に済ませたい | [[1103_Amazon Kinesis]] |
