# Amazon Kinesis

> [!info] 次々と届くデータを「届いたそばから」リアルタイムに受け取って流す土管。4種類の兄弟がいる。
> 用語が不安なら先に [[1102_データ分析の超基礎・用語集]] を。担当は分析5段階のうち**①収集**（リアルタイム）。
> 既存ノート [[0805_Kinesis]]（08_アプリケーション統合フォルダ）も参照。

## 超かんたんに言うと

センサーの値・アプリの操作ログ・動画など、**止まることなく次々と発生するデータ**を、ためてから処理するのではなく**発生したそばから受け取って流す**ための仕組みです（＝[[1102_データ分析の超基礎・用語集|ストリーミング処理]]）。

> [!example] たとえ話
> 川の流れのように、データが絶え間なく流れてきます。Kinesis はその川に**取水口や水路を作る道具**。流れを止めずに、必要な場所へ水（データ）を引いていきます。

## 4種類の兄弟（用途で使い分ける）

Kinesis は1つの名前で、役割の違う4つのサービスがあります。

### ① Kinesis Data Streams（生のデータを流す水路）
データを**そのまま流す**いちばん基本のもの。流れてきたデータを [[1102_データ分析の超基礎・用語集|サーバーレス]]の処理（Lambda）などで受け取って自由に使えます。
> たとえ：川の流れそのもの。

### ② Amazon Kinesis Data Firehose（自動で運ぶベルトコンベア）★よく出る
流れてきたデータを、**[[0201_S3]] や [[1110_Amazon OpenSearch Service]] などへ自動で運んで保存**してくれる。
受け取る側の設定をするだけで、あとは勝手にためてくれるのがラク。管理不要（サーバーレス）。
> たとえ：ベルトコンベア。一定量たまったら自動で次の工程へ運ぶ。

### ③ Kinesis Data Analytics（流れる水をその場で集計）
流れているデータに対して、**その場で [[1102_データ分析の超基礎・用語集|SQL]] で集計や異常検知**をする。「直近5分の平均」「急増したら警告」など。
> 現在は「Amazon Managed Service for Apache Flink」という名前に変わっています。

### ④ Kinesis Video Streams（動画専用）
監視カメラやドライブレコーダーの**動画を受け取って保存・再生**する専用版。

## こんなときに使う（試験頻出シナリオ）

| やりたいこと | 答え |
|---|---|
| リアルタイムのログを [[0201_S3]] にためたい | **Firehose** |
| クリック操作を即座に分析したい | **Data Streams** + Lambda |
| センサーデータ → S3 → [[1109_Amazon Athena]] で分析 | **Firehose** → S3 |
| 流れるデータをその場でSQL集計したい | **Data Analytics**（Flink） |
| すでに Kafka を使っている | [[1104_Amazon MSK]] |

## Kinesis と [[1104_Amazon MSK]] の違い

どちらも「リアルタイムにデータを流す土管」ですが、出自が違います。

| | Kinesis | [[1104_Amazon MSK]] |
|---|---|---|
| 中身 | AWS独自の仕組み | [[1102_データ分析の超基礎・用語集|Apache Kafka]]（世界標準のOSS） |
| 管理 | 完全おまかせ（ラク） | やや管理が必要 |
| 向いている人 | AWSで一から始める人 | すでに Kafka 資産がある人 |
