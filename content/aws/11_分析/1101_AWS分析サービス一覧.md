# AWS分析サービス一覧

> [!info] このフォルダ（11_分析）の目次・全体像です。
> 用語が分からないときは、まず [[1102_データ分析の超基礎・用語集]] を読んでください。「データ分析って何をしてるの？」から説明しています。

## はじめに：分析サービスは「データを料理する道具」

データ分析は **「① 収集 → ② 蓄積 → ③ 加工 → ④ 分析 → ⑤ 可視化」** の5段階で進みます（料理と同じ流れ）。
AWSの分析サービスは、この各段階を担当する**専門の道具**です。1つで全部やる魔法の道具はなく、役割ごとに使い分けます。

| 段階 | やること | 料理でいうと | サービス |
|---|---|---|---|
| ① 収集 | データを集める（リアルタイム） | 食材をその都度配送 | [[1103_Amazon Kinesis]] / [[1104_Amazon MSK]] |
| ① 収集 | データを集める（外部購入） | 外部の仕入れ業者 | [[1105_AWS Data Exchange]] |
| ① 収集 | データを集める（定期コピー） | 定期スケジュール配送 | [[1112_AWS Data Pipeline]]（旧世代） |
| ② 蓄積 | ためたデータを管理する | 冷蔵庫の在庫管理 | [[1106_AWS Lake Formation]] |
| ③ 加工 | 使いやすい形に整える（手軽） | 自動野菜カッター | [[1107_AWS Glue]] |
| ③ 加工 | 超大量データを整える | プロ用の大厨房 | [[1108_Amazon EMR]] |
| ④ 分析 | その場でSQLで答えを出す | 注文どおり即調理 | [[1109_Amazon Athena]] |
| ④ 分析 | DWHで本格的に集計する | 専用の分析厨房でじっくり | [[1113_Amazon Redshift]] |
| ④ 分析 | ログを検索・分析する | 山積みの記録から一瞬で発見 | [[1110_Amazon OpenSearch Service]] |
| ⑤ 可視化 | グラフ・ダッシュボードにする | お皿に美しく盛り付け | [[1111_Amazon QuickSight]] |

> [!note] データの実体はどこにある？
> 上の表のデータは、ほとんどの場合 [[0201_S3]]（AWSの保管庫）にためられています。分析サービスはその [[0201_S3]] のデータを読み書きして仕事をします。

## よくある組み合わせパターン（実務・試験で頻出）

サービスは単独でなく**つなげて**使います。代表的な3パターンを、流れで覚えましょう。

### パターン1：リアルタイムでログを分析する
アプリのログを集めて、ためて、すぐ検索したい場合。
```
アプリ → Kinesis Firehose → S3 → Athena → QuickSight
（記録が出る  集めて運ぶ      ためる  SQLで集計  グラフ化）
              （即検索したい分は） ↘ OpenSearch（その場で検索・可視化）
```

### パターン2：データレイク基盤を作る
何でもためる「データレイク」を作り、いろんな道具で分析する場合。
```
いろんなデータ → Glue で整える → S3（データレイク）← Lake Formation が権限管理
                                     ↓
                              Athena / EMR で分析
                                     ↓
                                QuickSight で可視化
```

### パターン3：超大量データをまとめて処理する（バッチ）
夜間にペタバイト級のデータを一気に加工する場合。
```
S3 → EMR（Sparkで大規模加工）→ S3 → Athena → QuickSight
```

## どれを選ぶ？（迷ったときの早見チャート）

```
データを分析したい
│
├─ データを「届いたそばから」リアルタイムに集めたい？
│   ├─ AWSだけで完結させたい → Kinesis
│   │   ├─ S3やOpenSearchへ自動で運んでほしい → Firehose
│   │   ├─ 生のデータを流すだけ           → Data Streams
│   │   └─ 流れるデータをその場で集計      → Data Analytics（Flink）
│   └─ もともとKafka（カフカ）を使っている → MSK
│
├─ S3にあるデータを、手軽にSQLで分析したい？ → Athena
│   └─ ただし「ペタバイト級のDWHで本格集計」「既存DWHとS3を結合」なら → Redshift / Redshift Spectrum
│
├─ データの「下ごしらえ（整形・変換＝ETL）」をしたい？
│   ├─ 手軽に・管理不要で → Glue
│   └─ 超大量データを本格的に → EMR
│
├─ データレイクの「誰が何を見られるか」をまとめて管理したい → Lake Formation
│
├─ 大量のログを検索・可視化したい → OpenSearch
│
├─ 経営層向けのグラフ・ダッシュボードを作りたい → QuickSight
│
└─ 外部の有料データ（気象・金融など）を買いたい → Data Exchange
```
