# Amazon QuickSight

> [!info] 分析した数字を、人が一目で分かるグラフ・ダッシュボードにする「盛り付け係」。
> 用語が不安なら先に [[1102_データ分析の超基礎・用語集]] を。担当は分析5段階のうち**⑤可視化**。

## 超かんたんに言うと

分析して出てきた答えも、数字の表のままでは経営者や現場の人には伝わりません。
QuickSight は、その数字を**棒グラフ・円グラフ・地図などのダッシュボード**にまとめ、**ブラウザで見られるレポート**にしてくれるサービスです。こうした「経営判断のための可視化ツール」を [[1102_データ分析の超基礎・用語集|BI]]（ビジネスインテリジェンス）と呼びます。

> [!example] たとえ話
> 分析の最後の「**お皿への盛り付け係**」。
> [[1109_Amazon Athena]] が料理人として[[1102_データ分析の超基礎・用語集|SQL]]でデータを取ってきて、QuickSight がそれを美しいグラフに盛り付けて配膳します。

## 知っておくと良い特徴

### SPICE（スパイス）という高速エンジン
データをあらかじめ**メモリ（高速な作業台）に取り込んでおく**ことで、大量データでもサッとグラフが出る仕組み。名前は「超高速・並列・メモリ内計算エンジン」の頭文字。

### ML Insights（AIによる気づき）
ボタンひとつで **異常の検知**（急に増えた／減った）や **将来予測**、さらに「**今月の売上は？**」と日本語っぽく質問できる機能（QuickSight Q）が使える。

### 課金が見る人にやさしい
レポートを**見る人は「見た回数（セッション）」ぶんだけ課金**。大勢が時々見るだけ、という使い方だと安く済みます。

## どこからデータを持ってくる？

[[1109_Amazon Athena]] / Amazon Redshift / [[0301_RDS]] / [[0201_S3]] など、さまざまな場所のデータをグラフ化できます。

## 定番の流れ

```
S3（データの置き場）→ Glue（説明書）→ Athena（SQLで集計）→ QuickSight（グラフ化）
```

## 埋め込み機能

作ったダッシュボードを、**自社のWebアプリの中に埋め込んで**、自分のサービスの顧客に見せることもできます（Embedded Analytics）。

## こんなときに使う（試験頻出シナリオ）

| やりたいこと | 答え |
|---|---|
| 経営層向けのダッシュボードを作りたい | **QuickSight** |
| [[1109_Amazon Athena]] の分析結果をグラフにしたい | **QuickSight** |
| 自社アプリにダッシュボードを埋め込みたい | **QuickSight** Embedded |
| 異常検知や予測つきのレポートが欲しい | **QuickSight** ML Insights |
