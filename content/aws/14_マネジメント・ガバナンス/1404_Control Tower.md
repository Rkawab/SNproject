# AWS Control Tower（SAA ★★）

> [!info] 一言で
> **Organizationsを使ったベストプラクティスなマルチアカウント環境を、ボタン一つで自動構築・統制する**サービス。

## たとえ話
- **Organizations** ＝ 本社機能の「素材」（自分で組み立てる）
- **Control Tower** ＝ 「組み立て済みスターターキット」。推奨設定で一気に整う。

会社を新しく立ち上げるとき、経理ルール・監査ログ・セキュリティ方針を一つずつ手で設定するのは大変。Control Towerは **それらを最初からセットで自動構築** してくれる。

## 中身（自動でセットアップされるもの）
- **AWS Organizations**（アカウント階層・SCP）
- **AWS Config**（構成のコンプラ評価）
- **AWS CloudTrail**（監査ログ）
- **IAM Identity Center（旧AWS SSO）**（一元ログイン）
- ログ集約用アカウント等

## 主要概念
| 用語 | 意味 |
|---|---|
| ランディングゾーン | 推奨設定済みの初期マルチアカウント環境 |
| ガードレール | ルール。**予防的（SCPで禁止）** と **発見的（Configで検知）** の2種 |
| アカウントファクトリ | 統制ルール付きで新規アカウントを量産する仕組み |

## SAA頻出パターン
> [!tip]
> - 「多数のアカウントを **統制ルール付きで素早く** 整備したい」→ Control Tower
> - 「ベストプラクティスなマルチアカウントの **初期構築を自動化**」→ Control Tower
> - 単に「請求をまとめる・SCPを当てる」だけなら → **Organizations** で十分（Control Towerは上位の自動化）

## Organizations との使い分け
- 細かく自分で組みたい・既存組織を運用 → **Organizations**
- ゼロから推奨構成で一気に立ち上げ → **Control Tower**（内部でOrganizationsを使う）
