# AWS CloudFormation（SAA頻出 ★★★）

> [!info] 一言で
> **インフラの設計図（テンプレート）を書くと、その通りに環境を自動で建ててくれる** IaC（Infrastructure as Code）サービス。

## たとえ話
家を建てるとき、毎回大工に口頭で指示すると言い間違い・作り忘れが出る。
CloudFormationは **詳細な設計図（テンプレート）を渡せば、同じ家を何度でも寸分違わず建てる** 仕組み。図面を捨てれば家もまとめて解体できる。

## 基本概念

| 用語 | 意味 |
|---|---|
| テンプレート | YAML/JSON で書いた構成定義（EC2, VPC, RDS…） |
| スタック | テンプレートから作られたリソース一式の管理単位 |
| 変更セット (Change Set) | 適用前に「何が変わるか」を事前確認する差分 |
| ドリフト検出 | テンプレートと実物の手動変更によるズレを検知 |
| 冪等性 | 同じテンプレートなら何度実行しても同じ結果 |

## 主な構成要素（テンプレート内）
- **Resources**（必須）：作るリソース本体
- **Parameters**：実行時に渡す可変値（インスタンスタイプ等）
- **Mappings**：リージョン別AMI等の対応表
- **Outputs**：作成後に出力する値（エンドポイント等）
- **Conditions**：条件付き作成

## SAA頻出ポイント
> [!tip] 出題パターン
> - 「同じ構成を **複数アカウント／複数リージョン** に展開」→ **StackSets**
> - 「大きな構成を再利用可能な部品に分割」→ **ネストスタック（Nested Stacks）**
> - 「適用前に影響を確認したい」→ **変更セット**
> - 「手動変更されていないか確認」→ **ドリフト検出**
> - 「インフラをバージョン管理・再現性を持たせたい」→ CloudFormation（IaC）

## 似たサービスとの違い
- **Terraform**：マルチクラウド対応のサードパーティIaC（CloudFormationはAWS専用）。
- **Elastic Beanstalk**：アプリのデプロイ環境を自動構築（内部でCloudFormation利用）。CFnより抽象度が高い。
- **Service Catalog**：CFnテンプレートを「承認済み製品」としてカタログ配布（[[Service Catalog]]）。
- **Proton**：コンテナ/サーバーレスのデプロイ標準化に特化。
