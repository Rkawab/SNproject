# AWS マネジメント＆ガバナンス サービス一覧（SAA対策）

> [!info] このノートの位置づけ
> SAA「マネジメント＆ガバナンス」分野の **全体俯瞰＆復習用** ノート。
> 頻出の重量級は個別ノートに分けてある（下記リンク）。CloudWatch / CloudTrail / Config は `10_モニタリング` の既存ノート参照。

## 全体像（たとえ話）

巨大なオフィスビル群（=AWSアカウント）のオーナーとして考えると整理しやすい。

| 役割 | サービス |
|---|---|
| 設計図から自動で建てる | CloudFormation / Proton / Service Catalog |
| 複数ビルをまとめる本社機能 | Organizations / Control Tower |
| 監視カメラ・温度計・操作記録 | CloudWatch / CloudTrail / Config |
| 顧問コンサル（無駄・危険の指摘） | Trusted Advisor / Compute Optimizer / Well-Architected Tool |
| 設備の遠隔メンテ係 | Systems Manager |
| ソフトのライセンス台帳 | License Manager |
| 運行情報・お知らせ | Health Dashboard |
| 正面玄関 / 通用口 | Management Console / CLI |

★★★=ほぼ必出 / ★★=よく出る / ★=たまに・捨て候補

## ① デプロイ・構成の自動化（IaC系）

### CloudFormation ★★★ → [[1402_CloudFormation]]
インフラの設計図（YAML/JSON テンプレート）を書くと、その通りに環境を自動構築する **IaC** サービス。スタック単位で管理、冪等性あり。StackSets で複数アカウント/リージョン展開。

### AWS Proton ★
コンテナ／サーバーレスアプリの **デプロイ標準化テンプレート** をプラットフォームチームが配る仕組み。「社内承認済みの建売プラン」。SAAでは軽め。

### AWS Service Catalog ★★
IT部門が承認した構成（製品）だけを並べた **カタログ**。利用者はそこから選ぶだけ＝セルフサービス＋統制。「許可された構成のみ展開させたい」→ これ。

---

## ② マルチアカウント管理・ガバナンス

### AWS Organizations ★★★ → [[1403_Organizations]]
複数アカウントを束ねる本社機能。**一括請求**・**OU**・**SCP（上限ガードレール）**。SCPは許可を与えるのではなく上限を絞る点に注意。

### AWS Control Tower ★★ → [[1404_Control Tower]]
Organizations + Config + CloudTrail + SSO を **自動セットアップ** してベストプラクティスなマルチアカウント環境（ランディングゾーン）を構築。ガードレール／アカウントファクトリ。

---

## ③ 監視・ログ・構成記録（※既存ノート）

### Amazon CloudWatch ★★★ → [10_モニタリング/CloudWatch](../10_モニタリング/01_CloudWatch.md)
メトリクス・ログ・アラームの監視センター。**状態の監視**。メモリ使用率は CloudWatch Agent が必要。

### AWS CloudTrail ★★★ → [10_モニタリング/CloudTrail](../10_モニタリング/02_CloudTrail.md)
誰が・いつ・どのAPIを叩いたかの **操作履歴（監査ログ）**。「Who did what」。

### AWS Config ★★ → [10_モニタリング/AWS_Config](../10_モニタリング/03_AWS_Config.md)
リソース設定の **変化を記録＋ルール違反を検知**。構成のコンプライアンス評価。

> [!tip] 三者の最重要引っかけ
> - CloudWatch = **状態**（性能数値・アラーム）
> - CloudTrail = **操作**（誰が何のAPIを叩いた）
> - Config = **構成**（設定が基準に沿っているか・その変遷）

---

## ④ 監視の可視化（OSS互換）

### Amazon Managed Grafana ★
可視化OSS **Grafana** のフルマネージド版。複数データソースを綺麗なダッシュボードに。

### Amazon Managed Service for Prometheus ★
コンテナ監視の定番OSS **Prometheus** のフルマネージド版。主にEKS/Kubernetesのメトリクス収集。収集=Prometheus、可視化=Grafana のコンビ。

---

## ⑤ 最適化・コンサル系

### AWS Compute Optimizer ★
機械学習でEC2/Lambda等の **適正サイズを提案**（ライトサイジング）。「服のサイズ診断」。

### AWS Trusted Advisor ★★
自動健康診断。**5観点**＝コスト最適化／パフォーマンス／セキュリティ／フォールトトレランス／サービス制限。全項目は **Business以上のサポートプラン** が必要。「今動いてる環境の自動チェック」。

### AWS Well-Architected Tool ★
設計を **5（6）本柱** のベストプラクティスで自己診断する質問票。「設計レビュー」。Trusted Advisor との違い＝Toolは設計レビュー、Advisorは稼働環境の自動チェック。

> [!note] Well-Architected の柱
> 運用性・セキュリティ・信頼性・パフォーマンス効率・コスト最適化（＋持続可能性）

## ⑥ 運用・その他

### AWS Systems Manager (SSM) ★★★ → [[1405_Systems Manager]]
サーバー群を遠隔で一元運用する万能ツール箱。**Session Manager**（キー・踏み台なしでEC2接続）／Parameter Store／Patch Manager／Run Command 等。

### AWS License Manager ★
商用ソフト（Oracle/SQL Server/Windows等）の **ライセンス使用を一元管理**、上限超過を防止。BYOL管理。

### AWS Health Dashboard ★
**Service Health**（AWS全体の障害状況）と **Your Account**（自分のリソースに関わる個別通知）。

### AWS Management Console ★
ブラウザでAWSをGUI操作する正面玄関。試験では深掘りされない。

### AWS CLI ★★
コマンドラインからAWSを操作。スクリプト化・自動化・CI/CD用。「GUIでなくスクリプトで自動操作」→ CLI/SDK。
