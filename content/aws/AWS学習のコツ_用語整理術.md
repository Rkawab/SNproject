# AWS学習のコツ｜用語の洪水で溺れないための整理術

> [!info] このノートの目的
> AWS SAA の勉強で「サービス用語が多すぎてパンクする」を防ぐための**考え方・記憶術**をまとめたもの。個別サービスの詳細は各カテゴリフォルダ（01〜12）を参照。

---

## 大前提：用語を「単語」で覚えない

初心者が挫折する最大の原因は、S3・EC2・VPC・IAM… を**単語帳のように1個ずつ暗記**しようとすること。
AWSサービスは「ある目的を達成するための部品」なので、**目的（カテゴリ）→ 部品**の順で覚えると一気に整理される。

---

## コツ1：頭の中に「6大カテゴリの引き出し」を作る

新しい用語が出てきたら、必ずどれかの引き出しに放り込む。これだけで迷子にならない。

| カテゴリ | 役割（たとえ） | 代表サービス |
|---|---|---|
| コンピューティング | 働く人（CPU/サーバー） | EC2, Lambda, ECS |
| ストレージ | 物置・倉庫 | S3, EBS, EFS |
| データベース | 整理された書類棚 | RDS, DynamoDB, Aurora |
| ネットワーク | 建物の配線・通路・受付 | VPC, Route53, CloudFront, ELB |
| セキュリティ | 警備員・入館証 | IAM, KMS, Security Group |
| 監視・管理 | 監視カメラ・点検記録 | CloudWatch, CloudTrail |

> [!tip]
> 新サービスを見たら毎回「これはどの引き出し？」と自問する。それだけで定着率が変わる。

---

## コツ2：「会社の建物（オフィスビル）」の比喩で全体像を持つ

AWSを**ひとつの会社のオフィスビル**だと思うと腹落ちする。

- **VPC** = 自社専用のビル（外部と仕切られた敷地）
- **サブネット** = ビル内のフロア分け（公開フロア／秘密フロア）
- **EC2** = そのフロアで働く従業員
- **Security Group** = 各部屋のドアの鍵（誰を通すか）
- **NACL** = フロアの入口ゲート（フロア単位の通行許可）
- **IAM** = 入館証の発行・権限管理（人事＋警備）
- **S3** = 敷地外にある巨大な貸し倉庫
- **ELB** = 受付の案内係（来客を空いてる窓口に振り分ける）
- **Route53** = ビルの住所・電話帳（名前 → 場所の変換）

> [!note]
> 新サービスを学ぶたびに「この会社のどの役割の人/モノ？」と置き換える癖をつけると、丸暗記が"理解"に変わる。

---

## コツ3：SAAは「暗記」より「使い分け（選択）」の試験

SAAは「○○とは何か」より、**「この要件なら、どのサービスを選ぶ？」**を問う形式が中心。
だから**似たもの同士の"比較セット"**で覚えるのが最強。

| 比較セット | ざっくり違い | 関連ノート |
|---|---|---|
| S3 vs EBS vs EFS | 倉庫 / PCに挿す外付けSSD / 共有ファイルサーバー | [[0202_S3とEBS・EFSの違い]] |
| RDS vs DynamoDB | きっちりした表計算 / 超高速メモ帳 | [[0301_RDS]] / [[0303_DynamoDB]] |
| ALB vs NLB vs GWLB | 用途別ロードバランサー | [[0503_ALB・NLB・GWLBの違い]] |
| スケールアップ vs スケールアウト | 1人を鍛える / 人数を増やす | [[0501_高可用性とスケーラビリティ]] |

> [!tip]
> 「単体で何か」より「Aと比べてBはどう違う・どっちを選ぶか」を意識すると、試験の問われ方とピタッと合う。

---

## コツ4：手を動かす（無料枠で実際に触る）

EC2を1台立てて、S3にファイルを置いて、IAMユーザーを作る。
**1回さわった用語は10回読むより記憶に残る**。まずは3大カテゴリ（EC2 / S3 / IAM）だけでも実機で触ると霧が晴れる。

---

## コツ5：覚える順番（土台 → 背骨 → 肉付け）

1. **IAM** … 全サービスの土台。権限が分からないと何も始まらない → [[0601_IAM基礎]]
2. **VPC + EC2** … AWSの背骨 → [[0101_EC2やVPCとは]] / [[0406_VPC応用機能]]
3. **S3** … 一番出る・一番使う → [[0201_S3]]
4. **RDS / DynamoDB** … データ層 → [[0301_RDS]] / [[0303_DynamoDB]]
5. あとは試験範囲（負荷分散・アプリ統合・監視 等）を埋めていく

---

## まとめ（1行で）

> [!success] 結論
> **「カテゴリの引き出し」に放り込み →「オフィスビルの役割」に例え →「比較セット」で使い分けを覚え → 無料枠で触る」**。
> 単語の丸暗記をやめて"理解"に変えれば、用語の数は怖くなくなる。
