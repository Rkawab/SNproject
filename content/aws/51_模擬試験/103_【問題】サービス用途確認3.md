# 【問題】サービス用途確認3（公式サンプル風・全10問）

## この問題集の目的

要件文から主要サービスの役割を選ぶドリル。101/102で扱っていない分野（コンテナ・分析・機械学習・移行など）も含め、13分野をひと通り確認する。

## 対象レベル

SAA入口。単語暗記ではなく、要件からサービスを選ぶ練習にする。

## 問題数

10問

## 第1問（Compute ｜ マネージドバッチ処理）
ある企業は、データファイルを処理する数千件のコンテナ化されたバッチジョブを実行したい。ジョブのキューイングと、必要な分だけのコンピューティングリソースのプロビジョニングをマネージドに任せ、運用負荷を抑えたい。最も適切なサービスはどれか。

- A. AWS Batch
- B. Amazon EC2 Auto Scaling
- C. AWS Lambda
- D. Amazon Lightsail

> [!success]- 答え：**A**
> AWS Batchはジョブキューとコンピューティング環境を管理し、バッチジョブの規模に応じてリソースを自動的にプロビジョニングする。
> - A：正解。大量のバッチジョブのキューイングとスケジューリングをマネージドで実行できる。
> - B：EC2の台数調整はできるが、ジョブキュー管理機能は持たない。
> - C：短時間のイベント駆動処理向けで、長時間・大量のバッチジョブ基盤としては第一候補ではない。
> - D：簡易な仮想サーバー／アプリ向けで、バッチジョブ管理用途ではない。
> **関連ノート**：[[0703_AWS_Batch]] / [[0702_Lambda・ECS・RDS・Beanstalk比較]]

## 第2問（Storage ｜ 共有ファイルストレージ）
複数のLinux EC2インスタンスから同時にマウントして読み書きできる共有ファイルシステムが必要である。容量はデータ量に応じて自動拡張し、ファイルサーバーの管理は避けたい。最も適切なサービスはどれか。

- A. Amazon EFS
- B. Amazon S3
- C. Amazon EBS
- D. AWS Storage Gateway

> [!success]- 答え：**A**
> EFSは複数のEC2から同時マウントできるフルマネージドなNFS共有で、容量が自動的に伸縮する。
> - A：正解。複数AZのEC2から共有でき、容量管理が不要。
> - B：オブジェクトストレージで、POSIXファイルシステムとしてのマウント共有用途ではない。
> - C：原則として単一EC2にアタッチするブロックストレージで、多数同時共有には向かない。
> - D：オンプレミスとAWS間のハイブリッドストレージ用途。
> **関連ノート**：[[0202_S3とEBS・EFSの違い]] / [[0103_EC2のストレージ]]

## 第3問（Database ｜ インメモリキャッシュ）
読み取りが非常に多いアプリケーションで、データベースの前段にマイクロ秒〜ミリ秒級で応答するインメモリキャッシュを置き、繰り返し参照されるデータの読み取り負荷とレイテンシを下げたい。最も適切なサービスはどれか。

- A. Amazon ElastiCache
- B. Amazon RDS Multi-AZ
- C. Amazon Redshift
- D. Amazon Neptune

> [!success]- 答え：**A**
> ElastiCache（Redis／Memcached）はインメモリのキャッシュ層を提供し、DBへの読み取り負荷とレイテンシを軽減する。
> - A：正解。頻繁に参照されるデータをキャッシュして高速応答できる。
> - B：可用性は高まるが、スタンバイは読み取り用キャッシュではない。
> - C：分析用データウェアハウス。
> - D：グラフデータベースであり、汎用キャッシュ用途ではない。
> **関連ノート**：[[0304_ElastiCache]]

## 第4問（Networking / CDN / LB ｜ グローバルな低遅延接続）
オンラインゲームのTCP/UDPトラフィックを、世界中のユーザーから固定のエニーキャストIPアドレスで最寄りのAWSエッジに取り込み、リージョンのアプリケーションへ低遅延でルーティングしたい。最も適切なサービスはどれか。

- A. AWS Global Accelerator
- B. Amazon CloudFront
- C. Amazon Route 53 加重ルーティング
- D. AWS Direct Connect

> [!success]- 答え：**A**
> Global Acceleratorは固定エニーキャストIPでトラフィックをAWSエッジに取り込み、最適経路でリージョンへ届ける。TCP/UDPの非HTTP系にも向く。
> - A：正解。固定IPとAWSグローバルネットワークで低遅延・高可用なルーティングを実現する。
> - B：HTTP/HTTPSの静的・動的コンテンツのキャッシュ配信が主目的。
> - C：DNSレベルの分散であり、固定IPでの経路最適化とは異なる。
> - D：オンプレミスとAWSの専用線接続サービス。
> **関連ノート**：[[0405_ネットワークとコンテンツ配信]]

## 第5問（Security / IAM ｜ 一時的なクロスアカウント権限）
別のAWSアカウントの利用者に、自社アカウント内のリソースを一時的な認証情報で操作させたい。長期のアクセスキーは配布したくない。最も適切な方法はどれか。

- A. IAMロールを作成し、相手に `sts:AssumeRole` で引き受けさせる
- B. 相手の人数分のIAMユーザーを作成し、アクセスキーを渡す
- C. ルートユーザーの認証情報を共有する
- D. セキュリティグループで相手アカウントを許可する

> [!success]- 答え：**A**
> クロスアカウントアクセスはIAMロールの引き受け（AssumeRole）で一時認証情報を使わせるのが基本。
> - A：正解。長期キーを配らず、最小権限の一時認証情報で操作させられる。
> - B：長期キーの配布・管理は漏えいリスクが高い。
> - C：ルートユーザー認証情報の共有は厳禁。
> - D：セキュリティグループは通信制御であり、AWS操作権限は付与しない。
> **関連ノート**：[[0602_IAMロール詳細]] / [[0601_IAM基礎]]

## 第6問（Management / Monitoring ｜ 構成変更の追跡と準拠評価）
AWSリソースの設定変更履歴を記録し、「暗号化されていないEBSボリューム」などの社内ルールへの準拠状態を継続的に評価したい。最も適切なサービスはどれか。

- A. AWS Config
- B. AWS CloudTrail
- C. Amazon CloudWatch
- D. AWS Trusted Advisor

> [!success]- 答え：**A**
> AWS Configはリソース構成の変更を記録し、ルール（Config Rules）で準拠状態を継続的に評価する。
> - A：正解。構成履歴とコンプライアンス評価に適している。
> - B：APIコールの監査証跡が主目的で、構成準拠の継続評価は専門外。
> - C：メトリクスとログの監視が中心。
> - D：ベストプラクティス点検サービスで、任意ルールの継続評価は行わない。
> **関連ノート**：[[1003_AWS_Config]]

## 第7問（Application Integration ｜ 非同期メッセージング）
注文サービスと配送サービスを疎結合にしたい。急なトラフィック増加時にもメッセージを失わずバッファし、コンシューマが1件ずつ取り出して処理する構成にしたい。最も適切なサービスはどれか。

- A. Amazon SQS
- B. Amazon SNS
- C. AWS Step Functions
- D. Amazon Kinesis Data Streams

> [!success]- 答え：**A**
> SQSはメッセージをキューに保持し、プロデューサーとコンシューマーを時間的に分離（バッファ）する。
> - A：正解。負荷の波を吸収し、コンシューマーが自分のペースで処理できる。
> - B：1対多のファンアウト通知（プッシュ）が主目的。
> - C：ワークフローのオーケストレーション用途。
> - D：大量ストリーミングデータの順序処理向けで、単純なジョブ分離では過剰になりやすい。
> **関連ノート**：[[0802_SQS]] / [[0801_アプリケーション統合サービス一覧]]

## 第8問（Analytics ｜ S3上データのアドホッククエリ）
S3に蓄積したCSVやParquetのログを、サーバーを管理せず標準SQLでアドホックに分析したい。スキャンした分だけ課金される仕組みが望ましい。最も適切なサービスはどれか。

- A. Amazon Athena
- B. Amazon Redshift
- C. Amazon EMR
- D. Amazon QuickSight

> [!success]- 答え：**A**
> AthenaはS3上のデータに対してサーバーレスで標準SQLクエリを実行し、スキャン量に応じて課金される。
> - A：正解。インフラ管理なしでS3データを直接クエリできる。
> - B：常設のデータウェアハウスで、クラスター管理と相応のコストが伴う。
> - C：Hadoop/Sparkの大規模分散処理基盤で、アドホックSQLには重い。
> - D：可視化（BI）ツールであり、クエリエンジンそのものではない。
> **関連ノート**：[[1102_データ分析の超基礎・用語集]] / [[1107_AWS Glue]]

## 第9問（Machine Learning ｜ 画像解析）
ユーザーがアップロードした画像から、物体・シーン・不適切コンテンツを自動検出したい。機械学習モデルの構築や訓練は行わず、マネージドなAPIで実現したい。最も適切なサービスはどれか。

- A. Amazon Rekognition
- B. Amazon Textract
- C. Amazon Comprehend
- D. Amazon SageMaker

> [!success]- 答え：**A**
> Rekognitionは画像・動画の物体検出、シーン認識、不適切コンテンツ検出などを学習済みAPIで提供する。
> - A：正解。モデル訓練なしで画像解析を実行できる。
> - B：ドキュメントからのテキスト・フォーム・表の抽出が専門。
> - C：自然言語テキストの分析（感情・エンティティ等）向け。
> - D：自前でMLモデルを構築・訓練・デプロイする基盤で、要件に対しては過剰。
> **関連ノート**：[[1301_AWS機械学習サービス一覧]]

## 第10問（Migration / Transfer ｜ オンプレからS3への継続転送）
オンプレミスのNFSファイルサーバーから大量のデータを、ネットワーク経由でS3へ自動・高速・定期的に同期したい。転送の暗号化と整合性チェックも任せたい。最も適切なサービスはどれか。

- A. AWS DataSync
- B. AWS Snowball Edge
- C. AWS Database Migration Service
- D. Amazon S3 Transfer Acceleration

> [!success]- 答え：**A**
> DataSyncはオンプレミスのNFS/SMB等とAWSストレージ間のデータ転送を自動化・高速化し、暗号化と整合性検証を備える。
> - A：正解。ネットワーク経由の継続的・定期的なファイル転送に適している。
> - B：回線が細い場合の物理デバイス輸送向けで、定期的なネットワーク同期用途ではない。
> - C：データベースの移行・レプリケーション専用。
> - D：S3アップロードを高速化する機能だが、オンプレのファイル同期を自動化するものではない。
> **関連ノート**：[[1502_DataSync vs Firehose vs Storage Gateway]] / [[1501_移行と転送サービス一覧]]
