# 【問題】SAA模擬試験レベル3（公式サンプル風・全10問）

## この問題集の目的

コスト最適化、データ移行、分析、運用統制を含む総合問題で本番形式に慣れる。

## 問題数

10問

## 第1問（S3ストレージコスト）
企業はS3にドキュメントを保存している。アクセス頻度は予測できず、数か月アクセスされないファイルもあれば、突然参照されるファイルもある。取り出し時の遅延を大きく増やさず、アクセスパターンの分析や手動移行なしでコストを最適化したい。最も適切なストレージクラスはどれか。

- A. S3 Intelligent-Tiering
- B. S3 Standardのみ
- C. S3 Glacier Deep Archive
- D. S3 One Zone-Infrequent Access

> [!success]- 答え：**A**
> Intelligent-Tieringはアクセスパターンに応じて階層を自動移動し、予測不能なアクセスに適している。
> - A：正解。手動分析なしでコストを最適化しやすい。
> - B：頻度が低いデータでコスト最適化できない。
> - C：取り出しに時間がかかり、突然参照される要件に合わない。
> - D：単一AZ保存で、可用性要件によっては不適切。
> **関連ノート**：[[0203_S3ストレージクラス]]

## 第2問（NATコスト削減）
プライベートサブネットのアプリケーションは、大量のデータをS3へ定期的に送信している。現在はNAT Gateway経由で通信しており、データ処理料金が高い。インターネットを経由せず、コストを下げたい。最も適切な方法はどれか。

- A. S3用Gateway型VPCエンドポイントを作成し、ルートテーブルを更新する
- B. NAT Gatewayをより大きなインスタンスタイプへ変更する
- C. Internet Gatewayをプライベートサブネットに直接関連付ける
- D. AWS Direct Connectを必ず導入する

> [!success]- 答え：**A**
> S3 Gateway Endpointを使うと、VPC内からS3へプライベートに接続でき、NAT Gateway経由のコストを避けられる。
> - A：正解。S3への大量通信のコスト最適化で頻出。
> - B：NAT Gatewayにインスタンスタイプはない。
> - C：プライベートサブネットの設計に反する。
> - D：オンプレミス接続要件がないため過剰。
> **関連ノート**：[[0409_IGW・NAT Gateway・VPCエンドポイント]]

## 第3問（大容量初回移行）
企業はオンプレミスのアーカイブデータ500TBをS3へ移行する。インターネット回線で転送すると数週間以上かかる。移行後は追加データだけをオンラインで同期したい。初回移行に最も適切なサービスはどれか。

- A. AWS Snowball Edge
- B. Amazon Kinesis Data Streams
- C. Amazon SQS
- D. AWS Config

> [!success]- 答え：**A**
> Snowball Edgeは物理デバイスを用いた大容量データ移行に適している。
> - A：正解。ネットワーク帯域が不足する初回大容量移行に向く。
> - B：リアルタイムストリーミング処理。
> - C：メッセージキュー。
> - D：設定管理サービス。
> **関連ノート**：[[1501_移行と転送サービス一覧]]

## 第4問（継続同期）
オンプレミスのファイルサーバーで日々生成されるファイルを、AWS上のS3へ継続的に同期したい。差分転送、スケジュール実行、転送検証をマネージドに行いたい。最も適切なサービスはどれか。

- A. AWS DataSync
- B. AWS Snowball Edgeのみを毎日発送する
- C. Amazon Route 53
- D. Amazon GuardDuty

> [!success]- 答え：**A**
> DataSyncはオンプレミスストレージとAWSストレージ間のオンライン同期を自動化する。
> - A：正解。継続同期や差分転送に適している。
> - B：日次運用には現実的でなく、継続同期ではない。
> - C：DNSサービス。
> - D：脅威検知サービス。
> **関連ノート**：[[1502_DataSync vs Firehose vs Storage Gateway]]

## 第5問（ログの取り込み）
アプリケーションはクリックストリームデータを継続的に生成する。データをほぼリアルタイムでS3へ配信し、後でAthenaで分析したい。サーバー管理を避け、配信先へのバッファリングと再試行も任せたい。最も適切なサービスはどれか。

- A. Amazon Kinesis Data Firehose
- B. AWS DataSync
- C. Amazon EFS
- D. AWS Backup

> [!success]- 答え：**A**
> FirehoseはストリーミングデータをS3、Redshift、OpenSearchなどへ配信するマネージドサービス。
> - A：正解。S3への継続的なストリーム配信に適している。
> - B：ファイル同期サービス。
> - C：共有ファイルシステム。
> - D：バックアップ管理サービス。
> **関連ノート**：[[1103_Amazon Kinesis]]

## 第6問（S3分析）
監査チームはS3に保存されたCloudTrailログを必要なときだけSQLで分析したい。クラスターを常時稼働させたくない。最も適切なサービスはどれか。

- A. Amazon Athena
- B. Amazon Redshift provisioned clusterのみ
- C. Amazon ElastiCache
- D. Amazon MQ

> [!success]- 答え：**A**
> AthenaはS3上のデータをサーバーレスにSQLクエリでき、必要なときだけ実行できる。
> - A：正解。CloudTrailログのアドホック分析に適している。
> - B：常時クラスター管理が必要になりやすい。
> - C：キャッシュサービス。
> - D：メッセージブローカー。
> **関連ノート**：[[1109_Amazon Athena]]

## 第7問（リソース準拠評価）
企業は、すべてのEBSボリュームが暗号化されていることを継続的に確認し、違反があれば検出したい。リソースの設定履歴も確認できる必要がある。最も適切なサービスはどれか。

- A. AWS Config
- B. AWS CloudTrailのみ
- C. Amazon CloudWatch Logsのみ
- D. AWS Artifact

> [!success]- 答え：**A**
> Configはリソース設定を記録し、ルールで準拠状態を評価できる。
> - A：正解。暗号化設定の継続評価と履歴確認に適している。
> - B：誰がAPIを呼んだかは追えるが、現在の準拠状態評価はConfigが主役。
> - C：ログ保存であり、設定準拠評価ではない。
> - D：監査文書の取得サービス。
> **関連ノート**：[[1003_AWS_Config]]

## 第8問（運用接続）
本番EC2インスタンスはプライベートサブネットにある。運用担当者はトラブルシューティングのためシェル接続が必要だが、踏み台サーバーの管理やSSHポートの公開は禁止されている。操作ログも残したい。最も適切な方法はどれか。

- A. AWS Systems Manager Session Managerを使用する
- B. パブリックサブネットに踏み台EC2を作り、秘密鍵を共有する
- C. EC2へElastic IPを付与してSSHを公開する
- D. NAT GatewayへSSH接続する

> [!success]- 答え：**A**
> Session Managerはインバウンドポートを開けず、IAM制御とログ記録を使ってEC2へ接続できる。
> - A：正解。踏み台不要、SSH非公開、監査性の要件に合う。
> - B：踏み台管理と鍵共有のリスクがある。
> - C：攻撃面を増やす。
> - D：NAT Gatewayは受信SSHの接続先ではない。
> **関連ノート**：[[1405_Systems Manager]]

## 第9問（データ保護）
金融系アプリケーションのS3バケットでは、一定期間オブジェクトを削除・上書きできないようにする必要がある。規制対応としてWORMに近い保持を実現したい。どの機能を使用すべきか。

- A. S3 Object Lock
- B. S3 Transfer Acceleration
- C. S3 Select
- D. S3イベント通知

> [!success]- 答え：**A**
> S3 Object Lockは保持期間中の削除や上書きを防ぎ、コンプライアンス要件に使える。
> - A：正解。WORM要件に適している。
> - B：アップロード高速化。
> - C：オブジェクト内データ抽出。
> - D：イベント発火機能であり、削除防止ではない。
> **関連ノート**：[[0205_S3のバージョン管理]]

## 第10問（総合コスト最適化）
企業はプライベートサブネットのアプリケーションからS3へ大量のログを送信し、ログは30日後にほとんど参照されなくなる。コストを下げつつ運用負荷を増やしたくない。どの対応を選ぶべきか。（2つ選択）

- A. S3 Gateway型VPCエンドポイントを使用する
- B. S3ライフサイクルルールで低頻度アクセスまたはアーカイブクラスへ移行する
- C. すべての通信をNAT Gateway経由に固定する
- D. すべてのログをS3 Standardに永久保存する

> [!success]- 答え：**A, B**
> S3への通信経路と保存階層の両方を最適化する。
> - A：正解。NAT Gateway経由のデータ処理コストを避けやすい。
> - B：正解。アクセス頻度低下後のストレージコストを自動で下げられる。
> - C：S3大量通信ではコスト増の原因になる。
> - D：長期保存コストが高くなる。
> **関連ノート**：[[0409_IGW・NAT Gateway・VPCエンドポイント]] / [[0204_S3のライフサイクル管理]]
