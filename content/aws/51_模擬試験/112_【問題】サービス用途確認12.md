# 【問題】サービス用途確認12（セキュリティ・運用・ガバナンス）

## この問題集の目的

AWSサービスの代表的な用途を、1問1論点で確認する。今回はセキュリティ、運用、マルチアカウント管理を中心に扱う。

## 対象レベル

SAA基礎。サービス名と「何を解決するものか」を結び付ける。

## 問題数

10問

---

## 第1問（Security / IAM ｜ 脆弱性管理）
EC2インスタンスとAmazon ECR内のコンテナイメージを継続的にスキャンし、既知のソフトウェア脆弱性を検出したいと考えています。
最も適したサービスはどれですか。

- A. Amazon Inspector
- B. Amazon GuardDuty
- C. Amazon Macie
- D. AWS Shield

> [!success]- 答え：**A**
> Amazon Inspectorは、EC2やECRコンテナイメージなどを対象とする脆弱性管理サービスです。
> - A：正解。既知の脆弱性や意図しないネットワーク露出を継続的に評価する。
> - B：API活動やネットワークなどを分析して脅威を検知するサービスであり、ソフトウェア脆弱性スキャンが主目的ではない。
> - C：S3内の機密データを発見・分類するサービス。
> - D：DDoS攻撃の緩和サービス。
> **着眼点（SAA）**：「EC2/ECR」「CVE」「脆弱性スキャン」ならInspector。
> **関連ノート**：[[1603_脅威検知・監査]]
> **詳しい解説**：[[112_【解説】サービス用途確認12#第1問（Security / IAM ｜ 脆弱性管理）]]

## 第2問（Security / IAM ｜ セキュリティ検出結果の集約）
複数のAWSアカウントでAmazon GuardDuty、Amazon Inspector、Amazon Macieなどが生成する検出結果を、一元的に確認してセキュリティ態勢を評価したいと考えています。
最も適したサービスはどれですか。

- A. AWS Security Hub CSPM
- B. Amazon Detective
- C. AWS Artifact
- D. AWS Audit Manager

> [!success]- 答え：**A**
> AWS Security Hub CSPMは、複数のセキュリティサービスの検出結果とセキュリティ標準への準拠状況を集約します。
> - A：正解。マルチアカウントのセキュリティ態勢を一元表示できる。
> - B：検出済みの事象について関係性や根本原因を調査するサービス。
> - C：AWSのコンプライアンスレポートや契約文書を取得するサービス。
> - D：監査証拠の収集と評価を支援するサービス。
> **着眼点（SAA）**：「複数サービスの検出結果を一元化」ならSecurity Hub CSPM。
> **関連ノート**：[[1601_セキュリティサービス一覧]] / [[1603_脅威検知・監査]]
> **詳しい解説**：[[112_【解説】サービス用途確認12#第2問（Security / IAM ｜ セキュリティ検出結果の集約）]]

## 第3問（Security / IAM ｜ セキュリティ事象の調査）
GuardDutyが不審なAPI操作を検出しました。担当者はCloudTrailやVPC Flow Logsなどの関連情報をたどり、影響を受けたリソースと根本原因を効率よく調査したいと考えています。
最も適したサービスはどれですか。

- A. Amazon Detective
- B. AWS Security Hub CSPM
- C. Amazon Inspector
- D. AWS Config

> [!success]- 答え：**A**
> Amazon Detectiveは、セキュリティ関連データの関係を可視化し、検出結果の調査と根本原因分析を支援します。
> - A：正解。発見された不審事象を「掘り下げて調べる」用途に合う。
> - B：検出結果の集約とセキュリティ態勢管理が中心。
> - C：ワークロードの脆弱性管理が中心。
> - D：AWSリソースの構成履歴と準拠状況を記録・評価する。
> **着眼点（SAA）**：「検知後の調査」「根本原因」「関係の可視化」ならDetective。
> **関連ノート**：[[1603_脅威検知・監査]]
> **詳しい解説**：[[112_【解説】サービス用途確認12#第3問（Security / IAM ｜ セキュリティ事象の調査）]]

## 第4問（Security / IAM ｜ 監査証拠の自動収集）
企業はPCI DSSに関する内部監査を定期的に実施しています。AWSリソースの構成やユーザー操作などの証拠を継続的に収集し、監査用の評価レポート作成を効率化したいと考えています。
最も適したサービスはどれですか。

- A. AWS Audit Manager
- B. AWS Artifact
- C. AWS CloudTrail
- D. Amazon CloudWatch

> [!success]- 答え：**A**
> AWS Audit Managerは、選択した監査フレームワークに沿って証拠を継続収集し、評価を整理します。
> - A：正解。監査証拠の収集・レビュー・レポート作成を支援する。
> - B：AWS自身の監査報告書や契約文書をダウンロードするためのサービス。
> - C：API操作履歴は重要な証拠の一つだが、監査全体の評価管理サービスではない。
> - D：メトリクス、ログ、アラームによる運用監視が中心。
> **着眼点（SAA）**：「自社環境の監査証拠を自動収集」ならAudit Manager。
> **関連ノート**：[[1606_コンプライアンス]]
> **詳しい解説**：[[112_【解説】サービス用途確認12#第4問（Security / IAM ｜ 監査証拠の自動収集）]]

## 第5問（Management / Monitoring ｜ EC2への安全な接続）
プライベートサブネット内のEC2インスタンスを運用管理したいと考えています。インバウンドのSSHポートを開けず、踏み台サーバーやSSH鍵の配布も避ける必要があります。
最も適した機能はどれですか。

- A. AWS Systems Manager Session Manager
- B. EC2 Instance Connect Endpoint
- C. AWS Direct Connect
- D. Amazon Inspector

> [!success]- 答え：**A**
> Session Managerは、Systems Managerを介してEC2へ対話的に接続でき、インバウンドポートや踏み台を不要にできます。
> - A：正解。IAMで接続権限を管理し、操作ログも記録できる。
> - B：プライベートIPへのSSH/RDP接続経路を提供できるが、SSH/RDP自体を使わないという要件にはSession Managerがより直接的。
> - C：オンプレミスとAWS間の専用線接続サービス。
> - D：脆弱性管理サービスであり、管理セッションを提供しない。
> **着眼点（SAA）**：「SSHポート・踏み台・鍵をなくす」ならSession Manager。
> **関連ノート**：[[1204_SSM_Session_Manager]] / [[1405_Systems Manager]]
> **詳しい解説**：[[112_【解説】サービス用途確認12#第5問（Management / Monitoring ｜ EC2への安全な接続）]]

## 第6問（Cost Optimization ｜ リソースの適正サイズ推奨）
EC2インスタンス、Auto Scalingグループ、EBSボリューム、Lambda関数などの利用状況を分析し、コストと性能の両面から適切なサイズへの変更候補を確認したいと考えています。
最も適したサービスはどれですか。

- A. AWS Compute Optimizer
- B. AWS Cost Explorer
- C. AWS Budgets
- D. AWS Cost and Usage Report

> [!success]- 答え：**A**
> AWS Compute Optimizerは、構成と利用メトリクスを分析してライトサイジング推奨を生成します。
> - A：正解。過剰または不足しているリソースの具体的な変更候補を示す。
> - B：コストや使用量の可視化・分析が中心。
> - C：予算しきい値に対する通知が中心。
> - D：詳細な請求・使用量データを出力する。
> **着眼点（SAA）**：「利用率を基に適正サイズを推奨」ならCompute Optimizer。
> **関連ノート**：[[1202_Compute_Optimizer]] / [[1701_コスト管理サービス一覧]]

## 第7問（Management / Monitoring ｜ 承認済み製品の配布）
中央IT部門が承認したCloudFormationテンプレートだけを、各開発チームがセルフサービスで選択してプロビジョニングできるようにしたいと考えています。
最も適したサービスはどれですか。

- A. AWS Service Catalog
- B. AWS CloudFormation
- C. AWS Organizations
- D. AWS Systems Manager

> [!success]- 答え：**A**
> AWS Service Catalogは、組織が承認したAWS製品をカタログとして配布し、利用者にセルフサービスで提供します。
> - A：正解。標準化された製品の一覧、アクセス制御、制約を管理できる。
> - B：テンプレートからリソースを構築する基盤だが、承認済み製品のカタログ提供が主目的ではない。
> - C：複数アカウントの一元管理とポリシー適用を行う。
> - D：AWSリソースの運用管理機能を提供する。
> **着眼点（SAA）**：「承認済みテンプレートをカタログ化して配布」ならService Catalog。
> **関連ノート**：[[1203_CloudFormationとServiceCatalog]]

## 第8問（Management / Monitoring ｜ ランディングゾーン）
新たにAWSを利用する企業が、Organizationsを基盤としてログ集約、アカウント発行、統制ルールを含むベストプラクティス準拠のマルチアカウント環境を迅速に構築したいと考えています。
最も適したサービスはどれですか。

- A. AWS Control Tower
- B. AWS Resource Access Manager
- C. AWS Trusted Advisor
- D. AWS License Manager

> [!success]- 答え：**A**
> AWS Control Towerは、ベストプラクティスに基づくランディングゾーンを自動設定し、マルチアカウント環境を継続的に統制します。
> - A：正解。アカウントファクトリーやコントロールを利用できる。
> - B：対応リソースをアカウント間で共有するサービス。
> - C：AWS環境に対するベストプラクティスの推奨を提示する。
> - D：ソフトウェアライセンスの使用状況を管理する。
> **着眼点（SAA）**：「ランディングゾーン」「アカウント発行」「統制」ならControl Tower。
> **関連ノート**：[[1404_Control Tower]] / [[1403_Organizations]]

## 第9問（Management / Monitoring ｜ アカウント間のリソース共有）
AWS Organizations内の複数アカウントで、Transit GatewayやVPCサブネットなどの対応リソースを共有し、アカウントごとに同じリソースを重複作成することを避けたいと考えています。
最も適したサービスはどれですか。

- A. AWS Resource Access Manager（AWS RAM）
- B. AWS PrivateLink
- C. VPCピアリング
- D. AWS IAM Identity Center

> [!success]- 答え：**A**
> AWS RAMは、対応するAWSリソースをアカウント間またはOrganizations内で安全に共有します。
> - A：正解。中央所有のリソースを複数アカウントから利用できる。
> - B：VPC間でサービスをプライベートに公開する仕組み。
> - C：2つのVPC間に非推移的なネットワーク接続を作る。
> - D：従業員のAWSアカウントへのSSOを管理する。
> **着眼点（SAA）**：「サブネットやTransit Gatewayを他アカウントと共有」ならAWS RAM。
> **関連ノート**：[[1403_Organizations]] / [[0406_VPC応用機能]]
> **詳しい解説**：[[112_【解説】サービス用途確認12#第9問（Management / Monitoring ｜ アカウント間のリソース共有）]]

## 第10問（Security / IAM ｜ 組織全体のファイアウォール管理）
Organizations配下の多数のアカウントに対して、AWS WAFルール、AWS Shield Advancedの保護、セキュリティグループポリシーなどを中央から一貫して適用・監査したいと考えています。
最も適したサービスはどれですか。

- A. AWS Firewall Manager
- B. AWS Network Firewall
- C. AWS WAF
- D. AWS Security Hub CSPM

> [!success]- 答え：**A**
> AWS Firewall Managerは、Organizations内の複数アカウントとリソースにセキュリティポリシーを中央適用します。
> - A：正解。複数種類の保護設定を組織横断で管理する。
> - B：VPCトラフィックを検査するマネージドネットワークファイアウォール。
> - C：個々のWebリソースに対するL7ルールを定義する。
> - D：検出結果とセキュリティ態勢を集約する。
> **着眼点（SAA）**：「Organizations全体へWAF等を中央適用」ならFirewall Manager。
> **関連ノート**：[[1604_ネットワーク防御]] / [[1403_Organizations]]
> **詳しい解説**：[[112_【解説】サービス用途確認12#第10問（Security / IAM ｜ 組織全体のファイアウォール管理）]]
