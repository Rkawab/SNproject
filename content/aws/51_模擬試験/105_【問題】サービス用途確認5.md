# 【問題】サービス用途確認5（公式サンプル風・全10問）

## この問題集の目的

要件文から主要サービスの役割を選ぶドリル。101〜104で扱っていない切り口（簡易VPS・専用線・脅威検知・イベント連携・サーバー移行・コスト可視化など）を中心に、13分野を横断する。

## 対象レベル

SAA入口。単語暗記ではなく、要件からサービスを選ぶ練習にする。

## 問題数

10問

---

## 第1問（Compute ｜ 簡易な固定料金サーバー）
小規模なブログとWordPressサイトを、わかりやすい固定月額で、最小限の構成手順で立ち上げたい。VPCやIAMの細かな設計は避け、手早く始めたい。最も適切なサービスはどれか。

- A. Amazon Lightsail
- B. Amazon EC2（フルカスタム構築）
- C. AWS Lambda
- D. Amazon ECS（Fargate）

> [!success]- 答え：**A**
> Lightsailは固定料金で仮想サーバー・DB・ネットワークをまとめて簡単に立ち上げられる、入門向けの簡易サービス。
> - A：正解。小規模サイトを手早く・予測可能な料金で構築できる。
> - B：柔軟だが、ネットワークやスケールの設計を自前で行う必要がある。
> - C：常時稼働のWebサイトをまるごと動かす用途には向かない。
> - D：コンテナ運用の知識が必要で、小規模ブログには過剰。
> **関連ノート**：[[0701_コンピューティングサービス一覧]]

## 第2問（Storage ｜ 誤削除・上書きからの保護）
重要なドキュメントをS3に保存している。誤った上書きや削除があっても、以前のバージョンへ簡単に復元できるようにしたい。最も適切な機能はどれか。

- A. S3バージョニング
- B. S3ライフサイクルルール
- C. S3 Transfer Acceleration
- D. S3イベント通知

> [!success]- 答え：**A**
> S3バージョニングはオブジェクトの複数バージョンを保持し、上書き・削除後も以前のバージョンへ復元できる。
> - A：正解。誤操作からの復元に直接対応する。
> - B：ストレージクラス移行や失効の自動化で、復元機能ではない。
> - C：アップロードの高速化機能。
> - D：オブジェクト操作を起点に処理を起動する通知機能。
> **関連ノート**：[[0205_S3のバージョン管理]]

## 第3問（Networking / CDN / LB ｜ 専用線接続）
オンプレミスのデータセンターとAWSの間で、インターネットを経由しない、帯域が安定した専用の物理接続を確立したい。大容量データの継続的な転送と一貫した低レイテンシが求められる。最も適切なサービスはどれか。

- A. AWS Direct Connect
- B. AWS Site-to-Site VPN
- C. Amazon CloudFront
- D. AWS Global Accelerator

> [!success]- 答え：**A**
> Direct ConnectはオンプレとAWS間に専用の物理接続を提供し、安定した帯域と一貫した低レイテンシを実現する。
> - A：正解。インターネットを経由しない安定した専用線接続に最適。
> - B：インターネット上の暗号化トンネルで、帯域・レイテンシはインターネット品質に依存する。
> - C：コンテンツのエッジ配信が主目的。
> - D：固定IPでのトラフィック高速化で、専用線そのものではない。
> **関連ノート**：[[0405_ネットワークとコンテンツ配信]]

## 第4問（Security / IAM ｜ 脅威検知）
AWSアカウント内の不審なAPIコールや、悪意あるIPとの通信、侵害された可能性のあるインスタンスの挙動を、機械学習を用いて継続的に検知したい。エージェント導入なしで始めたい。最も適切なサービスはどれか。

- A. Amazon GuardDuty
- B. AWS WAF
- C. AWS Config
- D. AWS CloudTrail のみ

> [!success]- 答え：**A**
> GuardDutyはVPCフローログ・DNSログ・CloudTrailなどを分析し、脅威をマネージドに検知する。
> - A：正解。エージェントレスで継続的な脅威検知を行える。
> - B：Webアプリ層の攻撃フィルタで、アカウント全体の脅威検知ではない。
> - C：構成評価が主目的で、脅威検知サービスではない。
> - D：証跡記録は行うが、それ自体は脅威の自動検知を行わない。
> **関連ノート**：[[1603_脅威検知・監査]]

## 第5問（Application Integration ｜ イベントルーティング）
SaaSアプリやAWSサービスから発生するイベントを、内容に応じたルールで複数のターゲット（Lambda・Step Functions・SQSなど）へ振り分けたい。スケジュール実行にも使いたい。最も適切なサービスはどれか。

- A. Amazon EventBridge
- B. Amazon SQS
- C. AWS Step Functions
- D. Amazon SNS

> [!success]- 答え：**A**
> EventBridgeはイベントバスとルールでイベントを内容に応じてルーティングし、スケジュール実行にも対応する。
> - A：正解。イベント駆動の振り分けとスケジューリングに最適。
> - B：メッセージのバッファリングで、内容ベースのルーティング機能は持たない。
> - C：ワークフローのオーケストレーション用途。
> - D：1対多のファンアウト通知で、ルールベースの振り分けはEventBridgeが適切。
> **関連ノート**：[[0804_EventBridge]] / [[0801_アプリケーション統合サービス一覧]]

## 第6問（Serverless ｜ データ変更への反応）
DynamoDBテーブルに新しい項目が追加・更新されるたびに、サーバーを管理せずに後続処理（通知や集計）を自動で実行したい。最も適切な構成はどれか。

- A. DynamoDB Streamsを有効化し、AWS Lambdaをトリガーする
- B. EC2を常時起動し、テーブルを定期ポーリングする
- C. RDSのトリガーで処理を実行する
- D. ユーザーが手動で変更を確認して処理する

> [!success]- 答え：**A**
> DynamoDB Streamsは項目の変更を順序付きで記録し、Lambdaを自動起動して後続処理を実行できる。
> - A：正解。変更イベント駆動でサーバーレスに処理できる。
> - B：常時起動とポーリングは運用負荷とコストが高い。
> - C：DynamoDBの変更検知にRDSトリガーは使えない。
> - D：手動確認は自動化の要件に反する。
> **関連ノート**：[[0303_DynamoDB]] / [[0708_Lambda]] / [[0702_Lambda・ECS・RDS・Beanstalk比較]]

## 第7問（Containers ｜ コンテナイメージの保管）
チームはDockerイメージをプライベートに保管し、脆弱性スキャンを行い、ECSやEKSからセキュアにプルしたい。最も適切なサービスはどれか。

- A. Amazon Elastic Container Registry（ECR）
- B. Amazon S3
- C. AWS CodeCommit
- D. Docker Hub の公開リポジトリ

> [!success]- 答え：**A**
> ECRはマネージドなプライベートコンテナレジストリで、イメージ保管・脆弱性スキャン・IAM連携のプルに対応する。
> - A：正解。AWS統合のセキュアなコンテナイメージ管理に最適。
> - B：汎用オブジェクトストレージで、コンテナレジストリ機能は持たない。
> - C：ソースコードのGitリポジトリで、イメージレジストリではない。
> - D：公開リポジトリはプライベート保管・IAM連携の要件に合わない。
> **関連ノート**：[[0904_ECS]] / [[0902_オーケストレーション_フルマネージド]]

## 第8問（Migration / Transfer ｜ サーバーのリホスト移行）
オンプレミスの多数の物理・仮想サーバーを、アプリケーションをほぼ変更せずにそのままAWSのEC2へリホスト（lift-and-shift）移行したい。継続的なレプリケーションで切替時のダウンタイムも抑えたい。最も適切なサービスはどれか。

- A. AWS Application Migration Service（MGN）
- B. AWS Database Migration Service（DMS）
- C. AWS DataSync
- D. Amazon S3 Transfer Acceleration

> [!success]- 答え：**A**
> Application Migration Service（MGN）はサーバーを継続レプリケーションし、最小ダウンタイムでEC2へリホストする。
> - A：正解。サーバーのlift-and-shift移行の標準サービス。
> - B：データベースの移行・レプリケーション専用。
> - C：ファイル・オブジェクトの転送向けで、サーバーまるごとの移行ではない。
> - D：S3アップロードの高速化機能。
> **関連ノート**：[[1501_移行と転送サービス一覧]]

## 第9問（Cost Optimization ｜ コストの可視化と分析）
過去のAWS利用コストをサービス別・タグ別に可視化し、傾向を分析して将来のコストを予測したい。最も適切なサービスはどれか。

- A. AWS Cost Explorer
- B. AWS Budgets のアラートのみ
- C. Amazon CloudWatch メトリクス
- D. AWS CloudTrail

> [!success]- 答え：**A**
> Cost Explorerは過去のコストをサービス別・タグ別などで可視化・分析し、将来予測も行える。
> - A：正解。コスト傾向の分析と予測に最適。
> - B：予算しきい値の通知が主目的で、傾向分析の可視化はCost Explorerが適切。
> - C：リソースの性能メトリクス監視で、コスト分析用途ではない。
> - D：API監査証跡で、コスト分析機能はない。
> **関連ノート**：[[1701_コスト管理サービス一覧]] / [[0102_EC2購入プラン]]

## 第10問（Machine Learning ｜ 音声合成）
ニュース記事やお知らせのテキストを、自然な音声に変換して読み上げる機能をアプリに追加したい。機械学習モデルの構築は行いたくない。最も適切なサービスはどれか。

- A. Amazon Polly
- B. Amazon Transcribe
- C. Amazon Comprehend
- D. Amazon Translate

> [!success]- 答え：**A**
> Pollyはテキストを自然な音声に変換するテキスト読み上げ（TTS）サービス。
> - A：正解。テキストから音声を生成する用途に最適。
> - B：音声をテキストに変換する（STT）逆方向のサービス。
> - C：テキストの感情・エンティティ分析向け。
> - D：テキストの言語翻訳サービス。
> **関連ノート**：[[1301_AWS機械学習サービス一覧]]
