# CloudFormation と Service Catalog の違い

## 概要

SAA-C03試験において、CloudFormation と Service Catalog は混同しやすいサービスです。
- **CloudFormation**: 環境を組み立てる設計図そのもの（Infrastructure as Code）
- **Service Catalog**: CloudFormation の上層に乗る仕組み（統制付きの配布管理）

---

## Service Catalog とは

### 定義
開発者向けの**セルフサービスポータル**を実現するAWSサービス。CloudFormation テンプレートをカタログに載せ、統制された形で配布・管理する。

### 主な機能
- **セルフサービス窓口**: 開発者が承認済みの構成を自分で払い出せる
- **バージョン・使用履歴の一元管理**: ガバナンス（組織統制）を実装
- **承認済み構成の配布**: CloudFormation テンプレートの上位管理層

### 出題頻度
- **低め（0〜1問見るかどうか）**
- CloudFormation ほど重要ではない
- キーワード判定だけで対応可能

---

## CloudFormation との位置づけ

| 観点 | CloudFormation | Service Catalog |
|------|----------------|-----------------|
| **役割** | 環境を組み立てる設計図（中身） | 設計図をカタログに載せて統制付きで配る（上位管理層） |
| **インターフェース** | JSON/YAML テンプレート | ポータル UI / セルフサービス |
| **対象ユーザー** | インフラ設計者 | 開発者（セルフサービス利用者） |
| **ガバナンス** | 基本的に含まない | ガバナンス機能を提供 |
| **重要度（SAA）** | 高い | 低い |

---

## 試験対策

### キーワード判定フロー

問題文に以下のキーワードが来たら、**Service Catalog の製品**を選択:
1. **「承認済み構成を、開発者にセルフサービスで払い出す」**
2. **「バージョン・使用履歴を一元管理（ガバナンス）」**

### よくある誤選択

❌ **CloudFormation テンプレート** を答えてしまう

理由: 「環境を作る設計図そのもの」は正しく見えるが、**それ単体には以下が無い**
- 開発者向けのセルフサービス窓口
- バージョン・使用履歴の一元管理
- ガバナンス機能

✅ **正解: Service Catalog の製品**

## 実装イメージ

```
┌─────────────────────────────────────────────┐
│        Service Catalog（上位管理層）         │
│  - ポータル UI                              │
│  - バージョン管理                           │
│  - アクセス制御                             │
│  - 使用履歴追跡                             │
└─────────────────────────────────────────────┘
               ↓
┌─────────────────────────────────────────────┐
│    CloudFormation テンプレート（中身）       │
│  - JSON/YAML で環境定義                    │
│  - EC2, S3, RDS など構成                   │
└─────────────────────────────────────────────┘
               ↓
┌─────────────────────────────────────────────┐
│   AWS リソース（EC2 インスタンス等）        │
└─────────────────────────────────────────────┘
```

## 参考

**「セルフサービス」「一元管理」の 2 語が Service Catalog の指名ワード**

これらが問題に出たら、他の選択肢は検討せず Service Catalog を選ぶ。
