# Trusted Advisor 概要

## Trusted Advisor とは

AWS Trusted Advisor は、**AWS環境全体を横断的に監視する健康診断ツール**。以下の5つのカテゴリで自動的にチェック・提案を行う。

## Trusted Advisor の5つのカテゴリ

| カテゴリ | 説明 | 例 |
|---|---|---|
| **セキュリティ** | セキュリティ設定の問題を検出 | 0.0.0.0/0 のセキュリティグループ、アクセスキーの長期未ローテーション |
| **パフォーマンス** | リソースの低効率を検出 | CPU使用率90%超で長期運用するEC2、オーバープロビジョニング（※具体的な型番まで提案するのは [[1202_Compute_Optimizer]]） |
| **フォールトトレランス** | 冗長性・可用性の問題を検出 | ELB配下が単一AZのみで稼働 |
| **サービス制限（クォータ）** | AWS API呼び出し制限の消費状況を監視 | VPC数、セキュリティグループ数など |
| **コスト最適化** | 無駄なコストを検出 | アイドル状態のELB、低使用率EC2、未使用EIP、RIの購入提案 |

## SAA（Solutions Architect Associate）での出題パターン

### よく問われる範囲

- Trusted Advisor の5本柱の定義（「◯◯の問題を検出するのはどれ？」）
- Trusted Advisor のチェック項目の役割分担

### 引っかけやすいポイント ⚠️

**Trusted Advisor ではなく他のサービスの領域:**

| 問題 | 正しいサービス | 理由 |
|---|---|---|
| **請求額が前月比10%以上増加** | **AWS Budgets / Cost Anomaly Detection** | Trusted Advisorはコスト最適化で"無駄を見つける"（アイドルリソース等）。月次比較の**閾値監視**はBudgets/Cost Anomaly Detection の役割 |
| **EC2インスタンスが故障していないか** | **EC2ステータスチェック / CloudWatch** | リアルタイムの**システムヘルスチェック**はTrusted Advisorではなく、EC2ステータスチェックまたはCloudWatch Events/Alarms |

## 各サービスの役割分担（SAA重要論点）

```
┌─────────────────────────────────────────────────────────┐
│ AWS監視・管理ツール の分け方                               │
├─────────────────────────────────────────────────────────┤
│ Trusted Advisor    → 定期的な「健康診断」（横断的チェック） │
│ CloudWatch         → リアルタイム「メトリクス監視」        │
│ AWS Budgets        → 「予算・コスト警告」（月次・閾値ベース） │
│ Cost Anomaly       → 「異常なコスト増加」自動検知          │
│ EC2ステータスチェック → EC2個別の「ハードウェア故障検知」   │
└─────────────────────────────────────────────────────────┘
```

## 学習ポイント

> **SAAで問われるのは「Trusted Advisorの5本柱」までで十分。**
> 
> 個別チェック項目の暗記ではなく、「このチェックはセキュリティ？パフォーマンス？」と役割を理解することが大事。
> 
> 特に「Trusted Advisor ↔ Budgets/CloudWatch ↔ EC2ステータスチェック」の**役割分担**は本当に問われるSAA論点。

## Compute Optimizer との違い（SAA頻出）

両者とも「EC2が低使用率（オーバープロビジョニング）」を指摘するため混同しやすいが、**深さ**が違う。

| 観点 | Trusted Advisor | Compute Optimizer |
|---|---|---|
| 範囲 | 5分野（セキュリティ/性能/可用性/制限/コスト） | **コンピュートのみ**（EC2/ASG/EBS/Lambda/Fargate） |
| 出すもの | 「低使用率です」と**指摘するだけ** | 「**○○に変えろ、月◯◯円減**」と**型番まで処方** |
| 仕組み | ルールベースの点検 | **機械学習**でメトリクス分析 |

> 「幅広い点検」→ Trusted Advisor、「ライトサイジング・最適な型番提案」→ **Compute Optimizer**。詳細は [[1202_Compute_Optimizer]]。
