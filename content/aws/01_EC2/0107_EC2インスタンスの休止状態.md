# EC2インスタンスの休止状態（Hibernate）

**休止（Hibernate）= メモリ（RAM）の内容をEBSルートボリュームに保存してから停止する機能**。
再開すると、OS起動からやり直すのではなく**停止直前の状態（実行中アプリ・メモリ内容）がそのまま復元**される。PCの「休止状態」と同じ。

## 停止・休止・終了の比較

| | 停止（Stop） | 休止（Hibernate） | 終了（Terminate） |
|---|---|---|---|
| メモリ（RAM）の内容 | ❌ 消える | ✅ **EBSに保存され復元** | ❌ 消える |
| EBSのデータ | ✅ 残る | ✅ 残る | ❌ 削除（デフォルト） |
| 再開後 | OSを起動し直す | **作業中の状態から再開** | 再開不可 |
| EC2の課金 | なし | なし | なし |
| EBSの課金 | あり | あり（メモリ保存分も含む） | なし |
| 用途 | 開発環境の夜間停止・コスト削減 | 分析途中・起動に時間がかかるアプリ | 検証後の使い捨て環境 |

## 休止の主な前提条件

- **ルートEBSボリュームが暗号化**されていること
- EBSルートにRAM分の空き容量が必要（RAMの内容を書き込むため）
- 対応インスタンスタイプ・対応OSであること（RAMサイズ上限あり）
- 休止状態の維持は最大60日まで

## 判断フロー

```
もう不要？            → 終了（Terminate）
作業状態を維持したい？ → 休止（Hibernate）
それ以外で一時停止     → 停止（Stop）
```

## SAA試験ポイント

| 問われ方 | 答え |
|---|---|
| 「メモリ上の状態を維持したまま停止し、すばやく再開したい」 | **休止（Hibernate）** |
| 「起動に時間のかかるアプリを事前ウォームアップしておきたい」 | **休止** |
| 休止の前提条件は？ | **暗号化されたルートEBS**＋RAM分の空き容量 |
| 停止中の課金は？ | EC2は無課金、**EBSは課金継続** |
