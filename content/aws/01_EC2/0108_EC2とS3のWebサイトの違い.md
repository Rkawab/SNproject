# EC2 vs S3 静的Webサイトホスティングの違い

- **EC2** = 仮想**サーバー**を借りて、自分でNginx等のWebサーバーを動かす（動的処理も可能）
- **S3** = ストレージにHTMLファイルを置き、**Static website hosting** を有効化するだけで公開（静的のみ）

---

### 実際に何が違うか

**S3の場合（ファイルを置くだけ）**

```
1. バケットにindex.htmlをアップロード
2. Static website hostingをON
3. → もうアクセスできる
```

サーバーの起動・設定・メンテナンスが**ゼロ**

**EC2の場合（サーバーを立てる）**

```
1. インスタンスを起動
2. OSにSSHでログイン
3. Nginxをインストール・設定
4. ファイルを配置
5. セキュリティグループを設定
6. → やっとアクセスできる
```

サーバーが**ずっと動き続けるのでコストもかかり続ける**

---

### コストの違い（月1,000PVの小さなサイトの場合）


| |EC2（t3.micro）|S3|
|---|---|---|
|月額目安|**約1,000〜2,000円**|**数円〜数十円**|
|理由|誰も見てなくてもサーバーが動いてる|アクセスがなければほぼ0円|

S3は**使った分だけ**なので、アクセスが少ないサイトほど圧倒的に安い

---

### 一言まとめ

> **EC2** →「自分でサーバーを管理したい・動的処理もしたい」人向け
> 
> **S3** →「HTMLを置いてとにかく公開したい・運用コストを最小にしたい」人向け

静的サイトならほぼS3一択、というのがAWSのベストプラクティスです。

---

## SAA試験ポイント

| 問われ方 | 答え |
|---|---|
| 静的Webサイトを最小コスト・最小運用で公開 | **S3 静的Webサイトホスティング** |
| 静的サイトを**HTTPS**で配信したい | S3単体では不可 → **CloudFront を前段に置く**（S3のWebサイトエンドポイントはHTTPのみ） |
| 静的サイトを世界中に高速配信 | **S3 + CloudFront** |
| サーバーサイド処理（PHP・Django等）が必要 | **EC2**（または Lambda 等） |
| S3静的ホスティングの必須設定 | Static website hosting 有効化＋**バケットポリシーで公開許可**＋インデックスドキュメント指定 |
