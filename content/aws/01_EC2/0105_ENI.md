# ENI（Elastic Network Interface）

**ENI = EC2にアタッチする仮想ネットワークインターフェース（仮想LANカード）**。
EC2インスタンス自体はネットワーク情報を持たず、**IPアドレス・MACアドレス・セキュリティグループはすべてENIに紐づく**。

```
[ENI]
  ├ プライベートIPアドレス（必須・固定）
  ├ Elastic IP（任意で関連付け）
  ├ セキュリティグループ
  └ MACアドレス
      ↓ アタッチ
[EC2インスタンス]
```

## 構造上の位置づけ

ENIは**サブネットの中**に作られる（＝特定のAZに属する）。

```
VPC
 ├ サブネットA（10.0.1.0/24）
 │    └ ENI①（10.0.1.10）── EC2-A
 └ サブネットB（10.0.2.0/24）
      └ ENI②（10.0.2.20）── EC2-B
```

## ENIの付け外しでできること

ネットワーク情報がENI側にあるため、**ENIを別のEC2に付け替えると、IPアドレス・MACアドレス・セキュリティグループが丸ごと移動**する。

```
EC2-A ← ENI（10.0.1.10）
   ↓ デタッチ → 別インスタンスへアタッチ
EC2-B ← ENI（10.0.1.10）   ※IPはそのまま
```

### 代表的なユースケース

| ユースケース | 内容 |
|---|---|
| **フェイルオーバー** | 障害時にENIを待機系EC2へ付け替え、同じIPでサービス継続 |
| **MACアドレス紐付けライセンス** | MACに紐づくソフトライセンスをENIごと移動して維持 |
| **複数ENIの付与（デュアルホーム）** | 1台のEC2に「公開用」「管理用」など複数ネットワークを持たせる |

## 補足事項

- すべてのEC2は**プライマリENI（eth0）**を必ず1つ持ち、これは**デタッチ不可**
- 追加できるENI数はインスタンスタイプによって異なる
- ENIの付け替えは**同一AZ内**でのみ可能（ENIはサブネット＝AZに属するため）

## SAA試験ポイント

| 問われ方 | 答え |
|---|---|
| 障害時に同じプライベートIPを引き継いで切り替えたい | **ENIを待機インスタンスに付け替え** |
| セキュリティグループは何に適用される？ | インスタンスではなく**ENI**に適用 |
| ENIは別AZのインスタンスに付け替えられる？ | **不可**（同一AZ内のみ） |
