# EC2のストレージ（インスタンスストア / EBS / EFS / FSx）

EC2にアタッチできるストレージの種類と使い分けを整理する。

| 種類 | 正式名称 | 一言でいうと |
|---|---|---|
| **インスタンスストア** | Instance Store | EC2に**物理直結**した一時ストレージ（停止で消える） |
| **EBS** | Elastic Block Store | ネットワーク接続の**永続ブロック**ストレージ（EC2の「Cドライブ」役） |
| **EFS** | Elastic File System | 複数EC2から同時マウントできる**共有ファイル**ストレージ（NFS） |
| **FSx** | Amazon FSx | 特定用途向けマネージドファイルシステム（Windows / Lustre等） |

> [!tip] 例え：PCのストレージで対応づけ
> - インスタンスストア → PC**内蔵**SSD（本体と一心同体。消えるのが特徴）
> - EBS → USB**外付け**SSD（取り外して別PCに付け替え可能）
> - EFS / FSx → 社内の**共有フォルダ（NAS）**（複数PCから同時アクセス）

## 1. インスタンスストア vs EBS vs EFS

| 項目 | インスタンスストア | EBS | EFS |
|---|---|---|---|
| ストレージ種別 | ローカルブロック | ネットワークブロック | ネットワークファイル（NFS） |
| データの保持 | ❌ 消える（Stop/Terminateで消失） | ✅ 永続的 | ✅ 永続的 |
| 速度 | ◎ 非常に高速（物理直結） | ○ 中〜高速 | △ 中速 |
| レイテンシ | ◎ 最も低い | ○ やや高い | △ EBSより高い |
| アタッチ対象 | ❌ そのEC2のみ（固定） | ❌ 1つのEC2（基本） | ✅ 複数EC2から同時マウント |
| AZ制約 | そのインスタンスのAZ固定 | ❌ 同一AZ内のみ | ✅ 複数AZにまたがれる |
| 容量管理 | ❌ 固定（変更不可） | △ 事前指定（後から拡張可） | ✅ 自動伸縮（指定不要） |
| バックアップ | ❌ 不可 | ✅ スナップショット（S3保存） | ✅ AWS Backup対応 |
| インスタンス間移動 | ❌ 不可 | ✅ デタッチ→別EC2にアタッチ | ✅ そもそも共有なので不要 |
| 対応OS | ○ 対応インスタンスのみ | ◎ Linux / Windows両対応 | ❌ **Linuxのみ** |
| 暗号化 | △ インスタンスタイプ依存 | ✅ KMS対応 | ✅ KMS対応 |
| コスト | ◎ 無料（インスタンス料金に込み） | ○ 安い（GB単価低） | ❌ 高い（EBSの約3倍） |
| 主な用途 | キャッシュ・一時データ・バッファ | OS・DB・本番データ | 共有コンテンツ・CMS・ホームDir |

> [!warning] インスタンスストアの鉄則
> **Stop / Terminate でデータが消える**（再起動Rebootでは消えない）。「失われてよい一時データ・キャッシュ」専用。永続化が必要なら EBS。

## 2. FSx — EFSで対応できない要件用

EFSはLinux専用（NFS）。**WindowsやHPC等の特殊用途には FSx** を使う。

### FSx for Windows File Server
- **Windows環境**の共有ストレージ（SMBプロトコル）
- **Active Directory（AD）と統合**できる
- 試験パターン：「Windows環境で複数EC2から共有ストレージ」→ **FSx for Windows**

### FSx for Lustre
- **超高速・超低レイテンシ**の並列ファイルシステム
- **S3と直接統合**できる（S3のデータを高速処理して書き戻す）
- 試験パターン：「機械学習・HPC・ビッグデータで高速ストレージが必要」→ **FSx for Lustre**

### FSx for NetApp ONTAP / OpenZFS
- エンタープライズNAS移行・ZFS移行用。SAA試験ではほぼ出ない → 今は無視でOK

## 3. 全体比較表

| 項目 | インスタンスストア | EBS | EFS | FSx for Windows | FSx for Lustre |
|---|---|---|---|---|---|
| データ永続性 | ❌ 消える | ✅ 永続 | ✅ 永続 | ✅ 永続 | ✅ 永続 |
| 速度 | ◎ 最速 | ○ 中〜高速 | △ 中速 | ○ 中〜高速 | ◎ 超高速 |
| 複数EC2から同時利用 | ❌ | ❌（基本） | ✅ | ✅ | ✅ |
| 対応OS | ○ 対応機種のみ | ◎ Linux/Windows | ❌ Linuxのみ | ❌ Windowsのみ | ❌ Linuxのみ |
| AZまたがり | ❌ | ❌ 同一AZのみ | ✅ | ✅ | ❌ 同一AZ |
| 容量管理 | ❌ 固定 | △ 手動 | ✅ 自動伸縮 | △ 手動 | △ 手動 |
| コスト | ◎ 無料 | ○ 安い | ❌ 高い | △ 中〜高 | △ 中〜高 |
| 主な用途 | キャッシュ・一時データ | OS・DB・本番 | Linux共有FS | Windows共有・AD連携 | ML・HPC・ビッグデータ |

## 4. SAA試験の判断フロー

```
共有ストレージが必要？
├── No
│   ├── 最高速の一時領域でよい？ → インスタンスストア
│   └── 永続化が必要 → EBS
└── Yes
    ├── Windows環境？ → FSx for Windows
    ├── 超高速・ML・HPC？ → FSx for Lustre
    └── それ以外のLinux共有？ → EFS
```

| 問題文のキーワード | 答え |
|---|---|
| 一時キャッシュ・最低レイテンシ・消えてよい | **インスタンスストア** |
| EC2のOS・DB用ディスク／スナップショットでバックアップ | **EBS** |
| **複数AZの複数EC2**から同時マウント（Linux） | **EFS** |
| Windowsファイル共有・AD統合 | **FSx for Windows** |
| HPC・機械学習・S3連携の高速ストレージ | **FSx for Lustre** |
