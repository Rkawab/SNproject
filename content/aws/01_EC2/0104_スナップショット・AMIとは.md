# AWS スナップショット と AMI まとめ

## スナップショットとは

**ある時点のEBSボリューム（ディスク）の中身をS3にコピーしたもの。**

```
EC2インスタンス（サーバー）
    └── EBSボリューム（ハードディスク）← これの中身をコピーしたものがスナップショット
```

### 主な特徴

| 項目 | 内容 |
|---|---|
| 保存先 | S3（直接アクセス不可） |
| バックアップ方式 | 増分（2回目以降は差分のみ） |
| 暗号化 | 暗号化ボリュームのスナップショットも暗号化される |
| クロスリージョン | コピー可能（DR用途） |
| クロスアカウント | 他のAWSアカウントと共有可能 |

### 主な用途

- バックアップ・リストア
- リージョン間コピー（災害対策）
- ボリュームのサイズ変更

### 復元のイメージ

> スナップショット（過去のディスクの状態）→ 新しいEBSボリュームを作成 → EC2にアタッチ

PCのCドライブをまるごとバックアップして、新しいHDDに書き込んでPCに接続するのと同じ概念。

---

## AMIとは

**サーバー（EC2）を丸ごと再現するための設計図。**  
スナップショット（ディスクの中身）＋インスタンス設定情報をセットで持つ。

```
AMI（サーバーの設計図）
  ├── インスタンスタイプ・OS・設定情報（容量はほぼゼロ）
  └── スナップショットへの参照（実データはスナップショット側）
```

> AMI自体はスナップショットのデータをコピーして持っているわけではなく、  
> 「このスナップショットを使え」と指し示すポインタ。  
> AMIを作ると、裏側でスナップショットも自動作成される。

---

## スナップショット vs AMI

| | 何のコピー？ | 復元すると？ | 主な用途 |
|---|---|---|---|
| **スナップショット** | ディスク（EBS）の中身 | ディスクが復元される | データバックアップ・引っ越し |
| **AMI** | サーバーの設定＋ディスク | サーバーまるごと復元される | サーバー複製・Auto Scaling |

## SAA試験ポイント

| 問われ方 | 答え |
|---|---|
| **Auto Scaling** で同じ構成のEC2を自動で増やしたい | **AMI**（起動テンプレートの元） |
| 「データのバックアップ」が欲しい | **スナップショット** |
| 「サーバーまるごと複製」したい | **AMI** |
| AMIを別リージョンで使いたい | AMIを**リージョン間コピー**してから起動（AMIはリージョン単位） |
| スナップショットのコストを抑えたい | **増分バックアップ**なので2回目以降は差分のみ課金 |
| 暗号化されていないスナップショットを暗号化したい | **コピー時に暗号化を指定**できる |

> [!info] AMIを作ると **スナップショットもセットで作られる**（実データはスナップショット側にある）。
