# Amazon MQ

> [!info]
> **ActiveMQ / RabbitMQ** のマネージド版メッセージブローカー。業界標準プロトコルに対応し、**既存システムをクラウドへ移行**する際の選択肢。新規開発なら [[0802_SQS]] / [[0803_SNS]] が推奨。

## たとえ話：昔ながらの社内便ルールをそのまま引っ越し

オンプレで長年使ってきた「社内便（既存のメッセージング基盤）」がある。
クラウドに移すとき、ルールごと作り変えるのは大変。
Amazon MQ なら**同じプロトコル（JMS/AMQP等）のまま**クラウドへ引っ越せるので、
アプリ側のコード変更を最小限にできる。

## 特徴

- 対応エンジン: **Apache ActiveMQ** / **RabbitMQ**
- 対応プロトコル（業界標準）:
  - **JMS**（Java Message Service）
  - **AMQP**, **MQTT**, **STOMP**, **OpenWire**, **WebSocket**
- ブローカーの構築・パッチ・運用をAWSが管理

## SQS / SNS との使い分け（SAA頻出）

| 要件 | 選ぶサービス |
|---|---|
| **新規開発**・AWSネイティブ・スケーラブル | [[0802_SQS]] / [[0803_SNS]] |
| **既存アプリ**が標準プロトコル(JMS/AMQP等)使用 | **Amazon MQ** |
| オンプレからコード変更**最小**で移行 | **Amazon MQ** |

> [!tip]
> キーワードは **「既存」「移行」「標準プロトコル」**。これらが無く単に「疎結合にしたい」なら SQS/SNS が正解。

## SAAポイント

- 「**既存のメッセージブローカー**（ActiveMQ/RabbitMQ）をクラウドへ」→ Amazon MQ
- 「アプリは **JMS/AMQP** を使用、**変更を最小限**に」→ Amazon MQ
- 単に「疎結合・スケーラブル」だけ → SQS/SNS（MQではない点に注意）
