# アプリケーション統合サービス一覧

AWSが提供する**アプリケーション間連携・メッセージング・認証**まわりのサービスをまとめる。マイクロサービス・サーバーレスアーキテクチャの構成要素として頻出。

| [[0803_SNS]] | Pub/Sub通知 | 1メッセージを全員に一斉配信 | 現行 |
| [[0804_EventBridge]] | イベントバス | 出来事をルールで仕分け・SaaS連携 | 現行 |

## 全体イメージ

```
[エンドユーザー]
       ↓ 認証
   [Cognito]
       ↓ JWT
  [API Gateway]
       ↓
   [Lambda / ECS]
       ├─→ [SQS] ─→ ワーカー処理
       ├─→ [SNS] ─→ 複数サービスに通知
       │            ├─→ [SES] メール送信
       │            └─→ [SQS] さらに分岐
       └─→ [Kinesis] リアルタイム分析
                      └─→ [S3 / Redshift] (Firehose)
```

## 使い分けの判断軸

### メッセージング系（SQS / SNS / Kinesis）

| 要件 | 選ぶサービス |
|------|------------|
| 1メッセージ = 1処理、順序より処理速度 | SQS Standard |
| 1メッセージ = 1処理、順序必須 | SQS FIFO |
| 1メッセージ → 複数サービスに同時配信 | SNS（+ SQS で Fan-out） |
| 大量データの**ストリーム**、複数コンシューマー、過去データ再生 | Kinesis Data Streams |
| ストリームを S3 等にひたすら配信 | Amazon Data Firehose |

### ワークフロー系

| 要件 | 選ぶサービス |
|------|------------|
| Lambda 複数のオーケストレーション | **Step Functions** |
| 人間の手作業を含む長時間ワークフロー | SWF（または Step Functions） |

### 認証系

| 要件 | 選ぶサービス |
|------|------------|
| アプリ利用者の認証 | Cognito（User Pools） |
| アプリ利用者に AWS リソース直アクセス権 | Cognito（Identity Pools） |
| AWS 操作するメンバー管理 | IAM |

## SAA頻出パターン

- **疎結合**と書いてあれば → SQS or SNS
- **Fan-out / 複数の購読者** → SNS + 複数SQS
- **イベント内容で分岐 / SaaS起点 / 定期実行(cron)** → EventBridge
- **複数Lambdaを順序・分岐・リトライで統括** → Step Functions
- **既存アプリ / 標準プロトコル(JMS,AMQP) / 移行** → Amazon MQ
- **SaaS↔AWSをノーコード連携** → AppFlow
- **GraphQL / モバイルのリアルタイム・オフライン同期** → AppSync
- **リアルタイム** + **ストリーミング** → Kinesis
- **サーバーレスAPI** → API Gateway + Lambda + DynamoDB
- **モバイル認証** → Cognito
