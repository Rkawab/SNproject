# AWS 機械学習（AI）サービス一覧

> [!info] このフォルダ（13_機械学習）の目次・全体像です。
> AWSの「機械学習サービス」は、ほとんどが **学習済みAIをAPIで呼ぶだけ** のサービスです。MLの知識がなくても使えます。

## 大前提：2層構造を押さえる（SAA最重要）

| 層 | たとえ | 中身 | 代表 |
|---|---|---|---|
| **AIサービス層**（高レベル） | 出来合いの惣菜を買う | AWSが学習済み。APIを呼ぶだけ。ML知識**不要** | Comprehend, Rekognition, Polly 等 |
| **MLプラットフォーム層**（低レベル） | 自分で一から料理する | 自分でモデルを作る・学習・デプロイ | **SageMaker** |

> [!important] 問題文に「**独自の/カスタムモデル**」が出たら **SageMaker**。
> それ以外で「特定タスク（翻訳・読み上げ・画像認識…）をやりたいだけ」なら、対応する**専用AIサービス**を選ぶ。

---

## 📝 テキスト・言語系

### Amazon Comprehend（コンプリヘンド＝「理解する」）
**文章の意味を読み取るAI。**
- たとえ：レビュー文の山を渡すと「これは怒ってる客、これは満足」と仕分けする**読解担当者**。
- できること：感情分析（ポジ/ネガ）、キーフレーズ抽出、言語判定、固有表現（人名・地名）抽出。
- 派生：**Comprehend Medical**（カルテから病名・薬名を抽出）。
- SAA：「**大量の顧客レビューの感情を分析**」→ これ。

### Amazon Translate（トランスレート＝翻訳）
**ニューラル機械翻訳。** Google翻訳のAWS版。
- たとえ：多言語対応の**自動通訳者**。
- SAA：「アプリを**多言語対応**」→ Translate。Comprehendで言語判定→Translateで翻訳、も頻出。

### Amazon Kendra（ケンドラ）
**社内ドキュメント検索の賢いAI検索エンジン。**
- たとえ：「有給は何日？」と**自然な質問文**で聞くと、社内規定PDFから答えの箇所を返す**司書**。
- 普通のキーワード検索と違い、**質問の意図を理解**する（セマンティック検索）。
- SAA：「**社内マニュアル群に自然言語で質問できる検索**」→ Kendra。

---

## 🗣️ 音声系

### Amazon Polly（ポリー）
**テキスト→音声（読み上げ）。**
- たとえ：文章を渡すと自然な声で朗読する**アナウンサー**。
- SAA：「**記事を音声で読み上げ**」「アクセシビリティ向上」→ Polly。

### Amazon Transcribe（トランスクライブ＝書き起こし）
**音声→テキスト（文字起こし）。** Pollyの逆。
- たとえ：会議の録音を議事録にする**速記者**。話者の聞き分けも可能。
- SAA：「**コールセンターの通話を文字起こし**」→ Transcribe。→Comprehendで感情分析が王道。

### Amazon Lex（レックス）
**チャットボット／音声ボットを作る基盤。** Alexaの中身の技術。
- たとえ：注文を受け付ける**会話ロボの頭脳**。「意図（Intent）」と「スロット（必要情報）」を聞き出す。
- 組み合わせ：Lex（会話理解）＋Polly（声で返答）＋Lambda（処理実行）。
- SAA：「**対話型カスタマーサポートbot**」→ Lex。

---

## 👁️ 画像・文書系

### Amazon Rekognition（レコグニション＝認識）
**画像・動画の解析。**
- たとえ：写真を見て「犬、人が3人、ここに顔」と教える**目**。
- できること：物体検出、顔認識・顔比較、不適切コンテンツ検出、有名人認識、動画内の人物追跡。
- SAA：「**アップロード画像の不適切判定**」「**顔認証**」→ Rekognition。

### Amazon Textract（テキストラクト＝extract/抽出）
**書類から文字・表・項目を抽出（OCRの進化版）。**
- たとえ：請求書や申込書をスキャンすると「金額欄＝¥10,000」と**項目とセットで**データ化する**事務員**。
- ただのOCRと違い、**表やフォームの構造**を理解する。
- SAA：「**紙の請求書/申込書をデジタル化してDB登録**」→ Textract。

---

## 🔮 予測・専門系

### Amazon Forecast（フォーキャスト＝予測）
**時系列データから将来を予測。**
- たとえ：過去の売上表から「来月は1200個売れる」と予測する**需要予測担当**。
- 用途：在庫最適化、需要予測、リソース計画。
- SAA：「**過去の売上から将来の需要を予測**」→ Forecast。

### Amazon Fraud Detector（フラウド＝詐欺）
**オンライン詐欺・不正検知。**
- たとえ：怪しい注文や偽アカウント登録を見抜く**警備員**。
- 用途：不正取引、偽アカウント、なりすまし検出。
- SAA：「**オンライン決済の不正検知**」→ Fraud Detector。

---

## 🛠️ 自分で作る人向け（最重要）

### Amazon SageMaker（セージメーカー）
**機械学習モデルを「作る・学習・デプロイ」する統合開発環境。**
- たとえ：他が「出来合い惣菜」なら、SageMakerは**プロ仕様のフルキッチン**。食材選びから調理・盛り付けまで全部自分でやる。
- カバー範囲：データ準備 → モデル構築 → 学習 → デプロイ（推論エンドポイント） → 運用監視の**全工程**。
- SAA：「**独自MLモデルを構築・学習・本番デプロイ**」「データサイエンティストがモデル開発」→ SageMaker。**他のAIサービスでは要件を満たせない（独自モデルが必要）**ときの答え。

## SAA頻出：input/outputで覚える早見表

| 入力 → 出力 | サービス |
|---|---|
| テキスト → 意味・感情 | **Comprehend** |
| テキスト → 別言語テキスト | **Translate** |
| 質問文 → 文書から答え | **Kendra** |
| テキスト → 音声 | **Polly** |
| 音声 → テキスト | **Transcribe** |
| 会話 → 対話ボット | **Lex** |
| 画像/動画 → 認識結果 | **Rekognition** |
| 書類画像 → 構造化テキスト | **Textract** |
| 時系列 → 将来予測 | **Forecast** |
| 取引データ → 不正判定 | **Fraud Detector** |
| 自前データ → 独自モデル | **SageMaker** |

## よくある組み合わせパターン（試験頻出）

```
コールセンター音声 → Transcribe（文字起こし）→ Comprehend（感情分析）→ 苦情を自動検知
多言語チャット     → Comprehend（言語判定）→ Translate（翻訳）→ Lex（対話）→ Polly（音声返答）
紙の書類          → Textract（項目抽出）→ DynamoDB等に保存
```
