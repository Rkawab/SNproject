# AWS データ転送料金（egress課金の基礎）

> [!info] このノートの狙い
> 「転送コストを最小化する構成はどれか」というSAA頻出のコスト問題を、**出口ごとの単価**で解けるようにする。関連：[[0405_ネットワークとコンテンツ配信]]（Direct Connect/VPN）｜[[0409_IGW・NAT Gateway・VPCエンドポイント]]（NAT課金）

---

## 大原則：課金されるのは「外へ出るデータ（アウト/egress）」

AWSは基本、**AWSから外へ出ていく下り（アウトバウンド）データに課金**する。**入ってくる（インバウンド/イン）は原則無料**。そして同じ「外に出す」でも、**どの出口を通すかで単価が違う**——ここがコスト問題の決め手。

> [!warning] 直感に反する最重要ポイント
> 「大量配布ならインターネット/S3が一番安い」は**誤り**。**Direct Connect経由のアウトの方が、インターネット経由のアウトより単価が安い**。既にDirect Connectがあるなら、大量egressはそこを通す方が安上がり。（この思い込みで一度誤答した：[[509_【問題】外部サイト模擬試験9#第7問（Cost Optimization ｜ 大量データ配布の転送コスト最小化（S3配信））]]）

---

## 主なデータ転送料金（東京リージョン・2026年4月時点の目安）

| 経路 | データ転送料金の目安 | 備考 |
|---|---|---|
| **同一AZ内** | 無料 | 同じAZ内の通信 |
| **AZ間** | 約 0.01 USD/GB | 別AZへ |
| **リージョン間** | 約 0.09 USD/GB | 別リージョンへ |
| **インターネット → AWS（イン）** | **無料** | 受信は基本無料 |
| **AWS → インターネット（アウト）** | 約 **0.114 USD/GB**（最初の10TB/月） | 100GB/月まで無料。IGW自体の利用料は無料 |
| **NATゲートウェイ** | 約 0.062 USD/GB（＋時間料金 約0.062 USD/時） | 通したGB数ぶん処理課金が上乗せ |
| **CloudFront** | 約 0.114 USD/GB（最初の9TB/月） | 1,000万HTTP(S)まで無料枠あり。**オリジン→CloudFrontは無料** |
| **VPCピアリング** | 約 0.01 USD/GB | 同一AZ内は無料 |
| **Direct Connect** | 約 **0.041 USD/GB**（東京/大阪→日本のDXロケーション） | **インバウンドは無料**。契約により異なる |
| **Storage Gateway** | 約 0.01 USD/GB | オンプレ→AWSストレージへの転送 |
| **DataSync** | 約 0.0125 USD/GB | ベーシックモードの料金 |

※数値は目安（アジアパシフィック東京、2026年4月時点）。試験は「**相対的にどれが安い/高い**」で解けばよく、正確な金額暗記は不要。

## SAAでの解き方（相対比較で十分）

安い順のイメージ：**同一AZ内（無料）＜ AZ間・VPCピアリング（0.01）＜ Direct Connect（0.041）＜ NAT（0.062）＜ インターネットアウト・CloudFront（0.114）**

> [!tip] コスト問題の着眼点
> - **既にDirect Connectがある × 大量のegress** → **Direct Connect経由が最安**（インターネットより安い）。「既存DCがある」は**答えのヒント**であって撒き餌ではない。
> - **NATゲートウェイはGB課金**（時間料金＋処理量）。大量トラフィックを通す設計で出てきたら「**高い**」と反応する。VPCエンドポイント（ゲートウェイ型のS3/DynamoDB）で置き換えられないか疑う。
> - **インバウンド（受信）は基本無料**。「アップロードのコスト」を心配させる選択肢はたいてい引っかけ。
> - **同一AZ内は無料／AZ・リージョンをまたぐと課金**。可用性のためにAZをまたぐ設計はコストと表裏。
> - **オリジン → CloudFront は無料**。CloudFrontでキャッシュ配信するとオリジンのegressを削れる、が定番の節約策。
