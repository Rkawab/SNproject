# VPC応用機能(ピアリング / Flow Logs / Traffic Mirroring / Network Access Analyzer)

VPCの**接続・可視化・監査**に関わる4機能をSAA観点でまとめる。

>[!info] 全体像のたとえ
>VPCを**自社専用ビル**に例えると、サブネット=フロア、EC2=社員、SG=部屋のドア、ルートテーブル=廊下の案内板。今回の4機能は「**隣のビルとの渡り廊下**」「**入退館記録**」「**会話の盗聴(合法)**」「**間取り図の監査AI**」にあたる。

---

## 1. VPCピアリング(Peering)

**2つのVPC同士を1対1でプライベート接続し、互いのリソースがインターネットを経由せず内部IP(プライベートIP)で直接通信できるようにする機能。**

### たとえ
上の機能を身近に言い換えると、**隣のビルと専用の渡り廊下でつなぐ**ようなもの。別会社のビル(別アカウントのVPC)とも繋げる。

### 特徴
- **1対1接続**(A↔B、A↔Cは別契約が必要)
- **推移的ルーティング不可**:A↔B、B↔Cがあっても **A→Cは通れない**(B経由NG)
- **CIDR重複不可**:両側のIPアドレス帯がカブってるとダメ
- リージョン跨ぎOK(Inter-Region Peering)、アカウント跨ぎもOK
- 同AZ内なら追加料金なし、跨ぐと転送料

>[!warning] SAA頻出ポイント
>VPCが3つ以上で「メッシュ状」になるなら **Transit Gateway** が正解になりがち(ピアリングだと管理が爆発する)。シンプルに2VPCだけ繋ぐ/コスト重視ならピアリング。

関連: [[0401_IPアドレス・CIDR]]

## 2. VPC Flow Logs(フローログ)

**VPC/サブネット/ENIを通過する通信の"メタデータ"(送信元・宛先IP、ポート、プロトコル、許可/拒否など)を記録するログ機能。通信の中身そのもの(ペイロード)は記録しない。**

### たとえ
上の機能を身近に言い換えると、ビルの**入退館記録(防犯カメラの録画)**。「いつ・誰が・どの部屋に・どこから入ろうとしたか」を記録するが、**会話の中身は録音しない**。

### 特徴
- **ENI / サブネット / VPC** 単位で取得可能
- 記録内容:送信元IP、宛先IP、ポート、プロトコル、**ACCEPT/REJECT**、バイト数など
- 保存先:**CloudWatch Logs / S3 / Kinesis Data Firehose**
- **パケットの中身(ペイロード)は見えない**(メタデータのみ)

>[!tip] SAA頻出シナリオ
>- 「**通信が拒否された原因調査**」→ Flow LogsでREJECTを確認 → SG/NACLを見直す
>- 「**パケットの中身まで見たい**」→ Flow LogsではNG、**Traffic Mirroring** を選ぶ
>- セキュリティ監査・コンプライアンス要件で必須になりがち

## 3. VPC Traffic Mirroring(トラフィックミラーリング)

**ENIを流れるパケットを中身(ペイロード)ごと丸ごと複製し、別の分析用ENIへ転送する機能。IDS/IPSやパケット解析ツールに、通信を"そのまま"渡して詳細に調べられる。**

### たとえ
上の機能を身近に言い換えると、ビルの会話を**そのまま隣の監視室で同時に聞く盗聴器(合法版)**。郵便物のコピーをセキュリティ部に転送するイメージ。

### 特徴
- **パケットの中身(ペイロード)を丸ごとコピー**して別ENIへ送る
- 用途:**IDS/IPS(侵入検知)、ネットワーク解析、トラブルシュート**
- 送信元:**Nitro世代のEC2のENI** のみ(古いインスタンスは対象外)
- フィルター指定でミラーするトラフィックを絞れる
- 受信側で Suricata / Zeek などの解析ツールを動かす構成が定番

### Flow Logsとの違い

| 項目 | Flow Logs | Traffic Mirroring |
|---|---|---|
| 取れる情報 | メタデータのみ | パケット全体 |
| 用途 | 監査・通信可否調査 | 侵入検知・詳細解析 |
| コスト | 安い | 高い(帯域も食う) |
| 保存先 | CloudWatch / S3 | 別ENI(リアルタイム) |

## 4. Network Access Analyzer

**SG・NACL・ルートテーブル・IGW/TGWなどの設定"だけ"を静的に解析し、意図しない通信経路が存在しないかを洗い出す監査機能。実際の通信は一切流さない。**

### たとえ
上の機能を身近に言い換えると、ビルの**間取り図とドアの設定を全部読み込んで、「この部屋からあの部屋に行ける経路あるよ」と教えてくれるAI監査人**。実際に歩かなくても、設計図だけで侵入経路を発見してくれる。

### 特徴
- **到達可能性を静的に解析**(実トラフィックを流さず、SG / NACL / ルートテーブル / IGW / TGW 等の設定だけで判定)
- 「**意図しない経路**」を見つけるのが目的
- 例:「インターネットからRDSに到達できる経路はないか?」を一発チェック
- **Network Access Scope** で範囲を定義し、合致する経路を列挙

### 似た機能との使い分け(SAA頻出比較)

| サービス | 役割 |
|---|---|
| **Reachability Analyzer** | 「**A→Bは通じる?**」を1対1で確認(ピンポイント診断) |
| **Network Access Analyzer** | 「**意図しない経路がないか?**」をスコープで網羅監査 |
| **Flow Logs** | 「**実際に何が通った?**」事後ログ |
| **Traffic Mirroring** | 「**実際の中身は?**」リアルタイム複製 |

---

## まとめ:一行整理

| 機能 | 一言で |
|---|---|
| **VPCピアリング** | VPC同士の専用渡り廊下(1対1、推移ルーティング不可) |
| **Flow Logs** | 通信メタデータの録画(誰がどこと通信したか) |
| **Traffic Mirroring** | パケット中身の複製(IDSで詳細解析) |
| **Network Access Analyzer** | 設計図ベースの経路監査(意図しない穴を発見) |

>[!tip] SAA問題文の見分け方
>- 「コスト重視 / 2VPCのみ」 → **ピアリング**
>- 「3つ以上のVPC・メッシュ」 → **Transit Gateway**(ピアリングは✕)
>- 「通信拒否の原因を調べたい」 → **Flow Logs**
>- 「ペイロード解析 / IDS連携」 → **Traffic Mirroring**
>- 「特定経路の到達性確認」 → **Reachability Analyzer**
>- 「意図しない公開経路の網羅監査」 → **Network Access Analyzer**
