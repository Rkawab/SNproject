# プロトコルとポート番号

セキュリティグループ・NACLの設定やSAAの問題文を読むのに必要な、主要プロトコルとポート番号の早見表。

## 主要プロトコル比較

| 項目 | SSH | HTTP | HTTPS | NFS |
|---|---|---|---|---|
| 役割 | サーバー操作 | Web閲覧 | Web閲覧（暗号化） | ファイル共有 |
| ポート | 22 | 80 | 443 | 2049 |
| 対象 | 管理者 | ユーザー | ユーザー | サーバー同士 |
| 公開する？ | ❌ | ✅（→443へ誘導が基本） | ✅ | ❌ |
| セキュリティ | 厳しく制限（管理者IPのみ） | 基本公開 | 基本公開 | 内部のみ（EFS用） |

## ポート番号 早見表

| ポート | プロトコル | 用途 / 関連AWSサービス |
|---|---|---|
| 22 | SSH | EC2へのリモート接続（代替: SSM Session Manager） |
| 53 | DNS | Route 53（サービス名の由来） |
| 80 | HTTP | Web（ALBリスナー） |
| 443 | HTTPS | Web暗号化（ALB + ACM証明書） |
| 2049 | NFS | **EFSのマウント**（SGで2049を許可する） |
| 3306 | MySQL | RDS for MySQL / Aurora MySQL |
| 5432 | PostgreSQL | RDS for PostgreSQL / Aurora PostgreSQL |
| 1433 | SQL Server | RDS for SQL Server |
| 6379 | Redis | ElastiCache for Redis |
| 3389 | RDP | Windows EC2へのリモートデスクトップ |

## SAA試験ポイント

| 問われ方 | 答え |
|---|---|
| EC2からEFSをマウントできない | EFSのSGで**TCP 2049（NFS）**のインバウンドを許可 |
| WebサーバーのSG設定 | **80/443を0.0.0.0/0**に開放、**22は管理者IPのみ** |
| RDS(MySQL)に接続できない | RDSのSGで**3306**をアプリサーバーのSGから許可 |
| SSHポートを一切開けたくない | **SSM Session Manager** を使う |
