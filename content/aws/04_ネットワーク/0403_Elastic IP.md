# Elastic IP（EIP）

**Elastic IP = AWSが提供する固定のパブリック（グローバル）IPアドレス**。
アカウントに割り当てて、EC2やNAT Gateway等に関連付けて使う。

## なぜ必要か

EC2の通常のパブリックIPは**停止→起動すると変わる**。
Elastic IPを関連付ければ、**ずっと同じパブリックIPを使い続けられる**。

| | 通常のパブリックIP | Elastic IP |
|---|---|---|
| 停止→起動後 | **変わる** | 変わらない |
| 別インスタンスへの付け替え | 不可 | **可能**（障害時の切り替えに使える） |
| 料金 | インスタンス料金に込み | **保有しているだけで課金**（2024年〜） |

## 主な用途

- WebサイトやAPIの公開（DNSにIPを固定登録したい場合）
- ホワイトリスト連携（接続先に「このIPから来る」と登録が必要な外部システム）
- 障害対応（EIPを別のインスタンスに付け替えて即切り替え）

> [!info] 実体はENIへの関連付け
> EIPは正確にはインスタンスではなく**ENI**に関連付く → [[0105_ENI]]

## SAA試験ポイント

| 問われ方 | 答え |
|---|---|
| 再起動してもIPが変わらないようにしたい | **Elastic IP** |
| 障害時に同じパブリックIPで別インスタンスへ切り替え | **EIPの付け替え**（またはENIごと移動） |
| デフォルトの上限 | **1リージョンあたり5個**（緩和申請可） |
| EIPのコスト注意点 | **保有しているだけで課金される**ため、不要になったら解放する |
| 多数のEC2に固定IPは非現実的 → 代替は？ | 前段に**NLB（固定IP）**や**Global Accelerator**を置く |

関連: [[0404_Route53]]（そもそもDNSで解決できるならEIP不要なことも多い）
