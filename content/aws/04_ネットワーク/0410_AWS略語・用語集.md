# AWS頻出略語・用語集

このドキュメントは、AWS認定試験（特にSAA）の問題文に頻繁に出現する略語と用語をまとめたものです。試験中に「知らない単語」に遭遇した際の迅速な翻訳表として活用できます。

> [!tip] 役割分担
> - ここは **AWSサービスの略語**（DX / SG / MGN 等）担当
> - サービス名以外の**一般IT用語**（ワークロード・疎結合等）は → [[3003_問題文頻出用語辞典]]
> - 問題文の**読み方そのもの**は → [[3002_問題文の読み方（4ステップ分解）]]

## ネットワーク関連

| 用語/略語 | 正式名・意味 | ひとことで | 用途・ポイント |
|---|---|---|---|
| **DX** | AWS Direct Connect | オンプレ↔AWSの**専用線**（閉域） | インターネット非経由で安全・安定・低遅延な接続。VPNより信頼性が高い |
| **VPN (Site-to-Site)** | 仮想プライベートネットワーク | オンプレ↔AWSを**インターネット経由**で暗号化接続 | インターネットを通すためDXより低速・高遅延。コスト低い |
| **VGW** | Virtual Private Gateway | VPN/DXのAWS側の**出入口** | VPCに取り付けて、オンプレ側のカスタマーゲートウェイ(CGW)と接続 |

## セキュリティ関連

| 用語/略語 | 正式名・意味 | ひとことで | 用途・ポイント |
|---|---|---|---|
| **SG** | セキュリティグループ | **インスタンス単位**の防火壁（**許可のみ**） | ステートフル。初期状態は全て拒否、必要な通信だけ許可する |
| **NACL** | ネットワークアクセスコントロールリスト | **サブネット単位**の防火壁（**許可＋拒否**） | ステートレス。明示的な許可・拒否の両方を設定可能 |

## ロードバランシング関連

| 用語/略語 | 正式名・意味 | ひとことで | 用途・ポイント |
|---|---|---|---|
| **ALB** | Application Load Balancer | **L7(HTTP/HTTPS)** ロードバランサー | アプリケーション層での負荷分散。パス・ホスト名によるルーティング可能 |
| **NLB** | Network Load Balancer | **L4(TCP/UDP)** ロードバランサー・**超低遅延** | ネットワーク層での負荷分散。極めて高速・高スループット |

## VPCエンドポイント関連

| 用語/略語 | 正式名・意味 | ひとことで | 用途・ポイント |
|---|---|---|---|
| **エンドポイント（Gateway型）** | VPCエンドポイント（ゲートウェイ型） | **VPC内専用・無料** | S3, DynamoDB向け。VPCルートテーブルで設定。PrivateLinkなし |
| **エンドポイント（Interface型）** | VPCエンドポイント（インターフェイス型） | **プライベートIP・オンプレからも可** | SQS, SNS, EC2など多数のサービス対応。PrivateLinkで実装 |
| **PrivateLink** | ― | Interface型エンドポイントの仕組み | VPC内に**プライベートIP**を作成し、AWSサービスに**閉域接続** |

## 移行・復旧関連

| 用語/略語 | 正式名・意味 | ひとことで | 用途・ポイント |
|---|---|---|---|
| **MGN** | AWS Application Migration Service | サーバー(VM)を**AWSへ移行** | オンプレ or 別クラウドのVM ⇒ EC2への移行（リフト&シフト） |
| **DRS** | AWS Elastic Disaster Recovery | **災害復旧** | 大規模障害時のマルチリージョンフェイルオーバー・復旧。MGNと紛らわしい |
| **ARC** | AWS Application Recovery Controller | マルチリージョンの**確実なフェイルオーバー** | リージョン全体の障害対応。RTO/RPOを最小化 |

## セキュリティ・攻撃対策関連

| 用語/略語 | 正式名・意味 | ひとことで | 用途・ポイント |
|---|---|---|---|
| **WAF** | AWS Web Application Firewall | **XSS/SQLi** などアプリケーション層の攻撃対策 | ALB/CloudFront/API Gateway前段に配置 |
| **Shield** | AWS Shield | **DDoS**（大量トラフィック）対策 | 基本は無料（Standard）、Advanced版でDDoS対応チーム・コスト保護つき |
| **PII** | 個人識別情報（Personally Identifiable Information） | 住所・電話番号・クレジットカード番号など**個人を特定できる情報** | データ保護・暗号化の対象。Macie で自動検出可能 |

## パフォーマンス・可用性指標

| 用語/略語 | 正式名・意味 | ひとことで | 用途・ポイント |
|---|---|---|---|
| **RTO** | 目標復旧時間（Recovery Time Objective） | 復旧の**速さ**（業務再開まで何分か） | 短い＝より高可用性・高コスト |
| **RPO** | 目標復旧時点（Recovery Point Objective） | データ損失の**許容量**（何分分のデータが失われてもよいか） | 短い＝より多くのバックアップが必要・高コスト |

## 学習時のコツ

### 問題文に出現する文脈パターン

1. **「インターネット非経由」という条件** → **DX（専用線）** が解答（VPNではない）
2. **「インスタンスごとの細かい制御」** → **SG（セキュリティグループ）**
3. **「サブネット全体のルール」** → **NACL**
4. **「超低遅延」「金融取引」** → **NLB**
5. **「VPC内でプライベート接続」「オンプレからAWSサービスを利用」** → **Interface型エンドポイント + PrivateLink**
6. **「VM/サーバーをAWSに移す」** → **MGN**（復旧ではなく移行）
7. **「個人情報が含まれる」「PII」という言葉** → **暗号化・Macie・IAMポリシー** で制限

### 試験本番での活用法

問題を読み始めたら、まず：
1. 出現している略語を**この表で確認**して訳す
2. 「何をするか（移行？復旧？接続？）」を明確にする
3. 「制約条件」（低遅延・無料・高可用性など）と照合する

これだけで、問題文の60%の理解度が大幅に上がります。
